## 设计流程

### 完整设计流程

```
1. 发现阶段（Discovery）
   □ 理解需求
   □ 用户研究
   □ 竞品分析
   □ 定义问题

2. 定义阶段（Define）
   □ 用户画像
   □ 用户旅程地图
   □ 信息架构
   □ 功能优先级

3. 设计阶段（Design）
   □ 线框图
   □ 视觉设计
   □ 原型
   □ 迭代优化

4. 开发阶段（Development）
   □ 设计交付
   □ 开发支持
   □ 走查验收
   □ 调整优化

5. 测试发布（Test & Launch）
   □ 用户测试
   □ 问题修复
   □ 上线发布
   □ 数据监控
```

### 1. 需求分析

#### 需求收集

```
关键问题：

业务目标：
- 为什么要做这个功能？
- 期望达成什么目标？
- 成功指标是什么？

用户需求：
- 目标用户是谁？
- 解决什么问题？
- 用户痛点是什么？

技术限制：
- 技术栈是什么？
- 有哪些限制？
- 时间预算如何？
```

### 3. 线框图（Wireframe）

#### 低保真线框图

```
目的：
✓ 快速探索布局
✓ 聚焦功能和结构
✓ 快速迭代
✓ 低成本试错

特点：
- 灰度设计
- 占位内容
- 基本形状
- 无细节

工具：
- Figma（矩形+文本）
- Balsamiq
- 纸笔草图
```

#### 中保真线框图

```
特点：
- 真实内容
- 更准确的布局
- 组件雏形
- 基本交互

适用场景：
- 向团队展示
- 开发人员评估
- 用户测试
```

#### Figma线框图技巧

```
1. 创建线框图组件库
   - 输入框
   - 按钮
   - 导航
   - 卡片等

2. 使用灰度颜色
   #F5F5F5 - 背景
   #E0E0E0 - 边框
   #757575 - 文本

3. 使用占位符
   - [Logo]
   - [图片]
   - Lorem ipsum文本
```

### 4. 视觉设计

#### 设计准备

```
1. 建立设计文件结构
   项目名称/
   ├─ 🎨 设计系统
   │  ├─ 颜色
   │  ├─ 字体
   │  └─ 组件
   ├─ 📱 页面设计
   │  ├─ 首页
   │  ├─ 列表页
   │  └─ 详情页
   └─ 📋 原型

2. 设置网格
   - 12列网格
   - Gutter: 24px
   - Margin: 16px (移动), 48px (桌面)

3. 定义样式
   - 颜色样式
   - 文本样式
   - 效果样式
```

#### 设计交付清单

```
页面设计：
□ 所有页面状态
  - 默认状态
  - 加载状态
  - 错误状态
  - 空状态
  - 成功状态

□ 响应式设计
  - 移动端 (375px, 414px)
  - 平板 (768px, 1024px)
  - 桌面 (1280px, 1920px)

□ 交互状态
  - Hover
  - Active
  - Focus
  - Disabled

组件：
□ 组件库完整
□ 变体齐全
□ 文档说明
```

### 5. 原型设计

#### Figma原型功能

```
基本交互：
- On click → Navigate to
- On hover → Change to
- On drag → ...
- After delay → ...

动画设置：
- Instant（瞬间）
- Dissolve（溶解）
- Smart animate（智能动画）
- Move in/out（移入/移出）
- Push（推入）
- Slide in/out（滑入/滑出）

时长：
- Fast: 200ms
- Medium: 300ms
- Slow: 500ms

缓动：
- Ease out（推荐）
- Ease in
- Ease in and out
- Linear
```

#### 原型最佳实践

```
✓ 关键流程原型化
✓ 保持交互一致
✓ 添加动画增强真实感
✓ 测试所有交互
✓ 提供使用说明

❌ 不要原型化所有页面
❌ 不要过度动画
❌ 不要忘记错误状态
```

## Figma到代码

### 开发模式（Dev Mode）

```
Figma开发模式功能：

1. 代码生成
   - CSS代码
   - iOS代码（Swift/SwiftUI）
   - Android代码（Kotlin/Compose）

2. 检查属性
   - 尺寸
   - 间距
   - 颜色
   - 字体
   - 阴影

3. 资产导出
   - SVG
   - PNG（1x, 2x, 3x）
   - PDF

4. 标注
   - 间距测量
   - 规格说明
   - 实现笔记
```

### 命名规范

#### 图层命名

```
组件命名：
Component/Variant/State

示例：
Button/Primary/Default
Button/Primary/Hover
Button/Secondary/Disabled

页面命名：
[设备] 页面名称 - 状态

示例：
Mobile - Login
Desktop - Dashboard - Loading
Tablet - Product Detail
```

#### 导出命名

```
格式：
type-name-variant-size

示例：
icon-search-outline-24
img-hero-home-desktop
logo-primary-horizontal

规则：
- 小写字母
- 用连字符分隔
- 描述性
- 包含尺寸
```

### 切图与导出

#### SVG导出

```
适用：
- 图标
- Logo
- 简单图形

Figma设置：
1. 选中元素
2. 右侧面板 → Export
3. 格式选择 SVG
4. 点击 Export

优化：
- 移除不必要的组
- 合并路径
- 使用 SVGOMG 进一步压缩
```

#### PNG导出

```
适用：
- 照片
- 复杂图形
- 需要栅格化的元素

导出尺寸：
- 1x：标准
- 2x：Retina屏
- 3x：高分辨率

命名示例：
hero-image.png
hero-image@2x.png
hero-image@3x.png
```

### 设计规范文档

#### 标注工具

```
Figma内置：
- 测量工具（Opt+鼠标悬停）
- 注释（Comment）
- Dev Mode标注

插件：
- Redlines
- Measure
- Annotate
```

#### 规范文档内容

```markdown
# 页面规范

## 布局
- 容器最大宽度：1200px
- 左右内边距：移动 16px，桌面 48px
- 网格：12列，gutter 24px

## 间距
- 组件内：8px, 12px
- 组件间：16px, 24px
- 区块间：32px, 48px

## 颜色
- 主色：#3B82F6
- 成功：#10B981
- 错误：#EF4444

## 字体
- 字体族：Inter
- 标题：700粗体
- 正文：400常规
- 行高：1.6

## 交互
- 过渡时长：300ms
- 缓动：ease-out
- 悬停：略微放大+阴影
```

## 版本控制

### 文件版本管理

```
命名规范：
项目名_版本号_日期.fig

示例：
Dashboard_v1.0_20260127.fig
Dashboard_v1.1_20260201.fig
Dashboard_v2.0_20260215.fig

版本号规则：
- 主版本：重大变更
- 次版本：功能增加
- 修订号：小改动

Figma内置：
- 自动保存历史
- Version history
- 恢复到特定版本
```

### 分支策略

```
Figma分支（Branch）：

用途：
- 探索新想法
- 大型改版
- 实验性功能

流程：
1. 创建分支：File → Create branch
2. 命名：feature/new-homepage
3. 独立工作
4. Review changes
5. Merge回主文件（手动）

最佳实践：
✓ 分支用于大改动
✓ 小改动直接在主文件
✓ 定期合并避免冲突
✓ 删除不用的分支
```

## 最佳实践

### 文件组织

```
Figma文件结构：

📁 项目名称
  ├─ 📄 00-Cover（封面）
  ├─ 📄 01-Design System（设计系统）
  │   ├─ Colors
  │   ├─ Typography
  │   ├─ Components
  │   └─ Icons
  ├─ 📄 02-Wireframes（线框图）
  ├─ 📄 03-Desktop Design（桌面设计）
  ├─ 📄 04-Mobile Design（移动设计）
  ├─ 📄 05-Prototypes（原型）
  └─ 📄 06-Archive（归档）

页面命名：
- [状态] 页面名
- 示例：[WIP] Homepage
- 示例：[Review] Dashboard
- 示例：[Final] Login
```

### 组件管理

```
组件库结构：

Components/
├─ Atoms/（原子）
│  ├─ Button
│  ├─ Input
│  └─ Icon
├─ Molecules/（分子）
│  ├─ Form Field
│  ├─ Search Bar
│  └─ Card Header
└─ Organisms/（有机体）
   ├─ Navigation
   ├─ Card
   └─ Modal

命名规范：
- Component/Variant/State
- 使用斜杠分组
- 描述性命名
```
