## 组件设计原则

### 1. 一致性

```
统一的组件行为：
✓ 相同类型的交互使用相同组件
✓ 组件外观在整个应用中保持一致
✓ 遵循已建立的设计模式
```

### 2. 可复用性

```
设计可复用组件：
✓ 通过变体(variants)处理不同场景
✓ 使用props/参数控制样式
✓ 避免硬编码内容
```

### 3. 可访问性

```
确保组件可访问：
✓ 键盘导航支持
✓ 屏幕阅读器兼容
✓ 适当的 ARIA 属性
✓ 足够的触摸目标大小
```

### 4. 状态管理

```
组件应有清晰的状态：
✓ Default（默认）
✓ Hover（悬停）
✓ Active/Pressed（激活/按下）
✓ Focus（聚焦）
✓ Disabled（禁用）
✓ Loading（加载中）
✓ Error（错误）
✓ Success（成功）
```

## 按钮设计

### 按钮类型层级

```css
/* 1. 主按钮 (Primary) - 最高优先级 */
.btn-primary {
  background: #3B82F6;
  color: white;
  border: none;
  padding: 12px 24px;
  border-radius: 8px;
  font-weight: 600;
  font-size: 16px;
  cursor: pointer;
  transition: all 0.2s;

  background: #2563EB;
  transform: translateY(-1px);

.btn-primary:active {
  transform: translateY(0);
  box-shadow: 0 2px 4px rgba(59, 130, 246, 0.2);
}

/* 2. 次要按钮 (Secondary) - 中等优先级 */
.btn-secondary {
  background: transparent;
  color: #3B82F6;

.btn-secondary:hover {
```

## 总结

### 核心要点

**组件设计原则**
- 一致性：统一的行为和外观
- 可复用性：通过变体处理不同场景
- 可访问性：支持键盘和屏幕阅读器
- 状态完整：default, hover, active, focus, disabled

**关键组件**
- 按钮：清晰的视觉层级（primary, secondary, text）
- 表单：良好的焦点状态和错误提示
- 卡片：合适的阴影和悬停效果
- 导航：清晰的活动状态指示
