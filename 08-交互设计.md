## 交互设计原则

### 1. 可预测性

```
用户应该能预测交互结果：
✓ 使用标准模式和图标
✓ 提供清晰的视觉提示
✓ 保持一致的交互行为
✓ 避免意外的行为
```

### 2. 即时反馈

```
每个操作都应有明确反馈：
✓ 点击按钮：视觉变化
✓ 表单提交：加载指示
✓ 操作成功：确认消息
✓ 发生错误：错误提示
```

### 3. 容错性

```
允许用户犯错并轻松恢复：
✓ 提供撤销功能
✓ 确认危险操作
✓ 自动保存
✓ 清晰的错误信息
```

### 4. 效率优先

```
减少用户完成任务的步骤：
✓ 智能默认值
✓ 键盘快捷键
✓ 批量操作
✓ 记住用户偏好
```

## 微交互

### 什么是微交互？

```
定义：
围绕单一任务的小型交互动画

组成：
1. 触发器（Trigger）：启动交互
2. 规则（Rules）：确定发生什么
3. 反馈（Feedback）：让用户知道正在发生什么
4. 循环与模式（Loops & Modes）：元规则
```

## 动画与过渡

### 动画时长

```css
/* CSS 变量定义 */
:root {
  /* 快速 - 简单属性变化 */
  --duration-fast: 150ms;

  /* 标准 - 大多数交互 */
  --duration-normal: 300ms;

  /* 慢速 - 复杂动画 */
  --duration-slow: 500ms;

  /* 超慢 - 页面过渡 */
  --duration-slower: 700ms;
}

/* 使用示例 */
.transition-fast {
  transition: all var(--duration-fast);
}

.transition-normal {
  transition: all var(--duration-normal);
}
```

**使用指南：**
```
< 100ms：瞬时，几乎察觉不到
100-300ms：快速，流畅
300-500ms：标准，明显但不慢
500-1000ms：慢，引起注意
> 1000ms：非常慢，可能让用户不耐烦

推荐：
- 悬停效果：150-200ms
- 点击反馈：100-150ms
- 页面元素：300-400ms
- 页面过渡：400-600ms
```

### 缓动函数 (Easing)

```css
/* 预定义缓动函数 */
:root {
  /* 标准曲线 */
  --ease-in: cubic-bezier(0.4, 0, 1, 1);
  --ease-out: cubic-bezier(0, 0, 0.2, 1);
  --ease-in-out: cubic-bezier(0.4, 0, 0.2, 1);

  /* 特殊效果 */
  --ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);
  --ease-elastic: cubic-bezier(0.68, -0.6, 0.32, 1.6);
}

/* 使用场景 */

/* 元素进入 - 使用 ease-out */
.fade-in {
  animation: fadeIn 0.3s var(--ease-out);
}

/* 元素离开 - 使用 ease-in */
.fade-out {
  animation: fadeOut 0.3s var(--ease-in);
}

/* 来回移动 - 使用 ease-in-out */
.slide {
  transition: transform 0.3s var(--ease-in-out);
}

/* 弹跳效果 */
.bounce {
  animation: bounce 0.5s var(--ease-bounce);
}
```

**缓动函数选择：**
```
ease-out（推荐用于大多数 UI）：
- 快速启动，慢速结束
- 给人轻快、响应迅速的感觉
- 适合：按钮、菜单、对话框

ease-in：
- 慢速启动，快速结束
- 适合：元素离开屏幕

ease-in-out：
- 慢-快-慢
- 适合：位置移动、大小变化

linear：
- 匀速
- 适合：旋转、颜色变化
- 不推荐用于位置和大小
```

## 手势交互

### 移动端手势

```javascript
// 滑动检测
let startX = 0;
let startY = 0;

element.addEventListener('touchstart', (e) => {
  startX = e.touches[0].clientX;
  startY = e.touches[0].clientY;
});

element.addEventListener('touchend', (e) => {
  const endX = e.changedTouches[0].clientX;
  const endY = e.changedTouches[0].clientY;

  const diffX = endX - startX;
  const diffY = endY - startY;

  // 判断滑动方向
  if (Math.abs(diffX) > Math.abs(diffY)) {
    if (diffX > 50) {
      console.log('向右滑动');
    } else if (diffX < -50) {
      console.log('向左滑动');
    }
  } else {
    if (diffY > 50) {
      console.log('向下滑动');
    } else if (diffY < -50) {
      console.log('向上滑动');
    }
  }
});
```

### 拖拽

```javascript
let isDragging = false;
let currentX;
let currentY;
let initialX;
let initialY;
let xOffset = 0;
let yOffset = 0;

const dragElement = document.querySelector('.draggable');

dragElement.addEventListener('mousedown', dragStart);
document.addEventListener('mousemove', drag);
document.addEventListener('mouseup', dragEnd);

function dragStart(e) {
  initialX = e.clientX - xOffset;
  initialY = e.clientY - yOffset;
  isDragging = true;
  dragElement.style.cursor = 'grabbing';
}

function drag(e) {
  if (isDragging) {
    e.preventDefault();
    currentX = e.clientX - initialX;
    currentY = e.clientY - initialY;
    xOffset = currentX;
    yOffset = currentY;

    setTranslate(currentX, currentY, dragElement);
  }
}

function dragEnd(e) {
  initialX = currentX;
  initialY = currentY;
  isDragging = false;
  dragElement.style.cursor = 'grab';
}

function setTranslate(xPos, yPos, el) {
  el.style.transform = `translate3d(${xPos}px, ${yPos}px, 0)`;
}
```

## 性能优化

### 1. 使用 transform 和 opacity

```css
/* ✅ 好 - 硬件加速 */
.animated {
  transform: translateX(100px);
  opacity: 0.5;
}

/* ❌ 避免 - 触发重排 */
.slow-animated {
  left: 100px;
  width: 200px;
}
```

### 2. will-change

```css
/* 告诉浏览器将要动画的属性 */
.will-animate {
  will-change: transform, opacity;
}

/* 动画结束后移除 */
.animation-done {
  will-change: auto;
}
```

### 3. 减少动画元素

```
✓ 只对必要元素应用动画
✓ 使用 transform 代替 position
✓ 批量更新 DOM
✓ 避免在滚动时触发复杂动画
```

## 总结

### 核心要点

**微交互**
- 提供即时反馈
- 使用细腻的动画
- 引导用户注意力
- 增强用户满意度

**动画原则**
- 时长：150-300ms 最佳
- 缓动：ease-out 适合大多数 UI
- 性能：使用 transform 和 opacity
- 目的：每个动画都应有明确目的

**反馈机制**
- 视觉反馈：必需
- 音频反馈：可选
- 触觉反馈：增强体验
- 进度指示：长时间操作必需

**交互模式**
- 遵循平台规范
- 保持一致性
- 提供清晰反馈
- 允许撤销
