# 第 08 章：布局与栅格系统

> 目标：理解栅格系统的原理，掌握响应式布局设计，让界面在各种设备上都能良好呈现。

---

## 8.1 为什么需要栅格系统

栅格（Grid）是一套隐形的结构性规则，解决以下问题：

- 设计师 A 和设计师 B 的间距不统一
- 元素位置看起来"差不多"但其实不对齐
- 不知道该用多宽的内容区域
- 不同屏幕尺寸下布局混乱

**没有栅格**：元素像随机散落，看起来不整齐
**有了栅格**：所有元素有规律地排列，视觉上更专业

---

## 8.2 基础概念

### 栅格的三要素

```
┌──┬───────────────────────────────────────┬──┐
│  │  │           列（Column）          │  │  │
│  │  │                                 │  │  │
│  │  │   ←列宽（Column Width）→       │  │  │
│  │  │                                 │  │  │
│  │  │← 槽宽（Gutter）→│              │  │  │
│  │  │                                 │  │  │
└──┴───────────────────────────────────────┴──┘
  ↑
边距（Margin）
```

- **Column（列）**：内容排列的垂直轨道
- **Gutter（槽宽/间距）**：列与列之间的间隔
- **Margin（边距）**：内容区域与屏幕边缘的距离

---

## 8.3 移动端栅格规范

### iOS 设计规范（以 iPhone 14 Pro 为例）

```
设备尺寸：390px × 844px（2倍屏）

推荐栅格：
• 列数：4 列（简单布局）或 6 列（复杂布局）
• 边距：16px（两侧各 16px）
• 槽宽：16px

内容区宽度 = 390 - 16×2 = 358px
单列宽度（4列）= (358 - 16×3) / 4 = 77.5px
```

**常见设备尺寸**：

| 设备 | 逻辑分辨率 | 推荐边距 |
|------|-----------|---------|
| iPhone SE | 375px | 16px |
| iPhone 14 | 390px | 16px |
| iPhone 14 Plus | 428px | 20px |
| iPhone 14 Pro Max | 430px | 20px |
| iPad Mini | 744px | 24px |
| iPad Pro 11" | 834px | 32px |

### 移动端常见布局模式

**单列布局（最常见）**：
```
┌─────────────┐
│ [16px 边距]  │
│ 内容元素     │
│             │
│ 内容元素     │
│             │
│ [16px 边距]  │
└─────────────┘
```

**双列布局（产品列表）**：
```
┌─────────────┐
│ [卡片] [卡片]│  ← 2列，每列 = (全宽 - 边距×2 - 槽宽) / 2
│             │
│ [卡片] [卡片]│
└─────────────┘
```

**不对称布局**：
```
┌─────────────┐
│ [2份] │ [1份]│  ← 左宽右窄，或左窄右宽
└─────────────┘
```

## 8.4 Web 端 12 列栅格

Web 设计普遍使用 **12 列栅格**，因为 12 可以被 1、2、3、4、6 整除，灵活性强。

### 标准 12 列规范

```
常见 Web 设计宽度：1440px（设计稿）或 1280px

12 列栅格配置：
• 总宽：1200px 内容区（两侧各 120px 空白）
• 列数：12
• 槽宽：24px
• 列宽：(1200 - 24×11) / 12 = 78px

实际列宽 = 78px + 24px(槽) = 102px/列（含槽）
```

### 元素宽度计算

```
占 N 列的元素宽度 = 列宽 × N + 槽宽 × (N-1)

占 1 列：78px
占 2 列：78×2 + 24×1 = 180px
占 3 列：78×3 + 24×2 = 282px
占 4 列：78×4 + 24×3 = 384px
占 6 列：78×6 + 24×5 = 588px（半宽）
占 8 列：78×8 + 24×7 = 792px（2/3宽）
占 12 列：1200px（全宽）
```

### 常见 Web 页面布局

**营销页/落地页**：
```
│                    全宽 Header                   │
│                                                  │
│         居中 8列 核心内容（Hero Section）          │
│                                                  │
│  [4列特性]  [4列特性]  [4列特性]                  │
│                                                  │
│        8列 文章内容     │  4列 侧边栏              │
│                                                  │
│                    全宽 Footer                   │
```

**后台管理系统**：
```
│ 2列侧边栏 │           10列主内容区                │
│          │  [统计卡片] [统计卡片] [统计卡片]        │
│  导航菜单  │                                     │
│          │  [数据表格（全宽）]                    │
│          │                                     │
```

## 8.5 响应式设计

响应式设计（Responsive Design）让同一套界面适应不同屏幕尺寸。

### 断点（Breakpoints）

```
xs：< 576px      手机竖屏
sm：576 - 768px  手机横屏 / 小平板
md：768 - 992px  平板竖屏
lg：992 - 1200px 桌面小屏
xl：1200 - 1440px 桌面常规
xxl：> 1440px    大屏显示器
```

### 响应式布局策略

**内容优先（Content-first）**：
```
移动端（单列）          平板（双列）          桌面（三列）
┌────────────┐          ┌──────┬──────┐       ┌────┬────┬────┐
│   卡片     │          │ 卡片  │ 卡片  │       │卡片│卡片│卡片│
│   ───      │          │      │      │       │    │    │    │
│   卡片     │          │ 卡片  │ 卡片  │       │卡片│卡片│卡片│
│   ───      │          │      │      │       │    │    │    │
└────────────┘          └──────┴──────┘       └────┴────┴────┘
```

**导航响应式**：
```
移动端                      桌面端
┌─────────────────────┐    ┌───────────────────────────────────┐
│ [Logo]          ≡   │    │ [Logo]  首页  产品  案例  关于  联系│
└─────────────────────┘    └───────────────────────────────────┘
点击 ≡ 展开抽屉菜单          直接显示所有导航项
```

### 流式布局 vs 固定布局

```
固定布局：
  内容区固定 1200px，两侧留白随屏幕变化
  优点：设计稳定，不会因拉伸变形
  适用：内容型网站

流式布局（Fluid）：
  内容区用百分比，随屏幕比例变化
  优点：充分利用屏幕空间
  适用：数据密度高的管理系统
```

## 8.6 8pt 基础栅格

除了列栅格，还需要一套**间距栅格**，控制所有元素的位置和间距。

### 8pt 系统的好处

```
所有间距都是 8 的倍数：4, 8, 12, 16, 24, 32, 48, 64...

好处 1：开发人员可以直接使用，不需要四舍五入
好处 2：设计一致性，视觉更整洁
好处 3：与常见设备像素密度（1x、2x、3x）整除，不会出现半像素
```

### 间距使用指南

```
4px  → 图标与文字之间的微小间隙
8px  → 标签内左右 padding、紧凑列表行间距
12px → 表单项内部间距
16px → 标准内边距、卡片内 padding、列表行高
24px → 卡片之间间距、区块内元素间距
32px → 较大区块间隔
48px → 页面主要章节分隔
64px → 页面级别的大间距
```

## 8.7 安全区域（Safe Area）

移动端需要考虑刘海、圆角、底部导航等遮挡区域。

### iOS 安全区域

```
iPhone 14 Pro（动态岛）：
  顶部 Status Bar：59px（含动态岛区域）
  底部 Home Indicator：34px

内容区域 = 844 - 59 - 34 = 751px（真正可用的设计区域）
```

**设计规则**：
- 重要内容（按钮、核心文字）不能放在安全区之外
- 背景色可以延伸到边缘（出血）
- 底部 Tab Bar 要避开 Home Indicator

### Figma 中设置安全区

1. 使用社区的 iOS/Android 设备 Frame
2. 安装 "iOS 16 UI Kit" 等组件库
3. 在设计稿中保留 Status Bar 和 Home Indicator 的位置

---

## 8.8 常见布局错误

| 错误 | 表现 | 解决方案 |
|------|------|---------|
| 内容顶边 | 内容紧贴屏幕边缘 | 添加 16-20px 边距 |
| 间距不一致 | 卡片间距有时 8 有时 12 | 遵循 8pt 系统 |
| 忽略安全区 | 按钮被底部 Home Indicator 遮挡 | 添加安全区 padding |
| 对齐不精确 | 元素差 1-2px 没对齐 | 使用 Figma 对齐工具 |
| 内容过宽 | 桌面上单行文字超过 80 字 | 设置最大宽度 max-width |

## 本章小结

| 概念 | 要点 |
|------|------|
| 栅格三要素 | 列（Column）、槽宽（Gutter）、边距（Margin）|
| 移动端 | 4-6列，16-20px边距，16px槽宽 |
| Web 端 | 12列栅格，灵活分割1/2/3/4列内容 |
| 响应式 | 断点处调整列数，内容从单列到多列 |
| 8pt 系统 | 所有间距都是8的倍数，保证一致性 |
| 安全区 | 避免内容被状态栏/导航遮挡 |
