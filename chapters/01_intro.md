# 第 01 章：UI/UX 设计概论

> 目标：理解 UI 与 UX 的本质区别，了解设计行业全景，建立正确的设计思维起点。

---

## 1.1 什么是 UX（用户体验设计）

**UX（User Experience Design）** 是关于人如何与产品交互的整体感受。

> "用户体验涵盖终端用户与公司、其服务及产品之间交互的所有方面。"
> —— 雅各布·尼尔森（Jakob Nielsen），UX 领域奠基人

UX 设计师关注的核心问题：

- 用户是谁？他们的目标是什么？
- 用户完成任务时遇到了哪些障碍？
- 产品流程是否顺畅、高效、令人愉悦？

### UX 的五个层次（Garrett 模型）

```
┌──────────────────────────────────┐
│  表现层 Surface     ← 视觉设计    │
├──────────────────────────────────┤
│  框架层 Skeleton    ← 界面布局    │
├──────────────────────────────────┤
│  结构层 Structure   ← 信息架构    │
├──────────────────────────────────┤
│  范围层 Scope       ← 功能需求    │
├──────────────────────────────────┤
│  战略层 Strategy    ← 用户需求    │
└──────────────────────────────────┘
        抽象 ↑              具象 ↓
```

UX 设计师从**底层战略**出发，向上构建每一层。

---

## 1.2 什么是 UI（用户界面设计）

**UI（User Interface Design）** 专注于用户与产品交互的视觉界面——颜色、字体、按钮、图标、间距等所有可见元素。

UI 设计师关注的核心问题：

- 界面是否美观、一致、专业？
- 按钮、输入框等组件是否符合视觉规范？
- 视觉层级是否清晰引导用户注意力？

### UI 与 UX 的关系

```
UX（体验设计）
├── 用户研究
├── 信息架构
├── 流程设计
├── 线框图
└── UI（界面设计）← UI 是 UX 的一部分
    ├── 视觉设计
    ├── 组件设计
    └── 动效设计
```

**类比**：
- UX = 建筑师（规划空间布局、动线、功能）
- UI = 室内设计师（选材、配色、家具摆放）

---

## 1.3 常见职位对比

| 职位 | 核心技能 | 工具 | 侧重点 |
|------|---------|------|--------|
| UX 研究员 | 用户访谈、数据分析 | Dovetail、Excel | 理解用户 |
| UX 设计师 | 信息架构、原型 | Figma、Axure | 流程与结构 |
| UI 设计师 | 视觉设计、组件 | Figma、Sketch | 界面美观 |
| 产品设计师 | UI+UX 全栈 | Figma | 端到端设计 |
| 交互设计师 | 动效、微交互 | Principle、ProtoPie | 交互细节 |

> 国内市场现状：大多数公司招聘 **UI/UX 设计师**，要求同时具备两种能力。

---

## 1.4 设计师的日常工作

一个典型的产品设计流程：

```
1. 接收需求 → 理解业务目标
      ↓
2. 用户研究 → 访谈、问卷、竞品分析
      ↓
3. 定义问题 → 用户画像、痛点梳理
      ↓
4. 发散方案 → 草图、头脑风暴
      ↓
5. 制作原型 → 线框图 → 高保真稿
      ↓
6. 用户测试 → 收集反馈
      ↓
7. 迭代优化 → 输出最终设计稿
      ↓
8. 开发交付 → 标注、切图、验收
```

## 1.5 行业现状与学习方向

### 主流设计平台

| 平台类型 | 特点 | 代表产品 |
|---------|------|---------|
| 移动端 App | 触控交互、小屏约束 | 微信、支付宝、抖音 |
| Web 端 | 响应式、鼠标键盘 | 各类 SaaS 产品 |
| 小程序/H5 | 轻量、跨平台 | 微信小程序 |
| 桌面软件 | 复杂功能、专业用户 | Figma、Adobe |
| 智能硬件 | 嵌入式 UI | 手表、车机、智能家居 |

### 2024-2026 设计趋势

- **AI 辅助设计**：Figma AI、Midjourney 生图、设计自动化
- **设计系统化**：Token 驱动、多端适配
- **无障碍设计**：法规推动 A11y 成为必备技能
- **动效与 3D**：更丰富的视觉层次

## 1.6 学习心态建议

**误区 1**：UI/UX 就是让界面"好看"
> 好看只是 UI 的一部分，更核心的是**解决用户问题**。

**误区 2**：学会工具就学会设计
> Figma 只是画图工具，设计思维才是核心能力。

**误区 3**：设计是主观的，没有对错
> 设计决策可以用数据和用户研究来验证。

**正确心态**：
- 以用户为中心，而非以自我审美为中心
- 设计是解决问题的系统方法
- 持续迭代，没有"完美"的设计

## 本章实践作业

**作业 1**：找 3 款你常用的 App，分别写出：
- 你认为 UX 做得好的地方（1-2 点）
- 你认为 UX 做得差的地方（1-2 点）
- UI 视觉上令你印象深刻的地方

**作业 2**：在手机上完成一次购物流程，记录你遇到困惑或卡顿的每一个步骤。这就是 UX 研究的基本功——**观察与记录**。

## 本章小结

| 概念 | 要点 |
|------|------|
| UX | 用户与产品交互的整体体验，包含研究、架构、流程 |
| UI | 界面的视觉呈现，是 UX 的子集 |
| 设计流程 | 研究 → 定义 → 发散 → 原型 → 测试 → 迭代 |
| 核心价值 | 以用户为中心，用数据验证决策 |
