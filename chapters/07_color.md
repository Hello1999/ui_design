# 第 07 章：色彩理论

> 目标：理解色彩的基本原理、情绪语义和配色方法，为产品设计出专业的色彩方案。

---

## 7.1 色彩基础

### 色彩属性（HSB 模型）

UI 设计中最常用 **HSB/HSV** 模型描述颜色：

```
H - Hue（色相）：颜色的种类
    0° = 红色
    60° = 黄色
    120° = 绿色
    180° = 青色
    240° = 蓝色
    300° = 洋红/紫色
    360° = 红色（循环）

S - Saturation（饱和度）：颜色的纯度
    0% = 灰色（无彩色）
    100% = 最纯的颜色

B - Brightness（明度）：颜色的亮暗程度
    0% = 黑色
    100% = 最亮
```

**调色技巧**：
```
增加饱和度 → 颜色更鲜艳、活跃
降低饱和度 → 颜色更柔和、专业
增加明度   → 颜色更浅淡（Tint）
降低明度   → 颜色更深沉（Shade）
```

### 色温

```
冷色系（蓝绿紫）          暖色系（红橙黄）
──────────────            ──────────────
科技、冷静、专业           活力、温暖、食欲
Notion、支付宝、Twitter    麦当劳、美团、拼多多
```

## 7.2 色轮与配色方案

### 基本配色方案

**1. 单色配色（Monochromatic）**
```
使用同一色相的不同明度/饱和度

示例（蓝色）：
  #EBF5FF → #90CDF4 → #4299E1 → #2B6CB0 → #1A365D

优点：和谐统一，专业感强
适用：简约风格产品、科技类界面
```

**2. 邻近色配色（Analogous）**
```
使用色轮上相邻（30-60°）的颜色

示例：蓝色 + 青色 + 绿色（各30°相邻）

优点：自然舒适，视觉统一
适用：自然、户外、健康类产品
```

**3. 互补色配色（Complementary）**
```
使用色轮上对角（180°）的颜色

示例：蓝色（#4299E1）+ 橙色（#ED8936）

优点：对比强烈，视觉冲击力强
注意：大面积使用会产生视觉疲劳，通常一个主色+少量互补色作为强调
```

**4. 三角配色（Triadic）**
```
使用色轮上等距（120°）的三种颜色

示例：红 + 黄 + 蓝

优点：丰富多彩，平衡感强
适用：儿童产品、游戏界面
```

**5. 分裂互补配色（Split-Complementary）**
```
主色 + 互补色两侧的颜色

示例：蓝色 + 橙红色 + 橙黄色

优点：保有对比感，但比互补色更柔和
适用：大多数 UI 产品的推荐方案
```

## 7.3 UI 配色系统

### 色彩角色分类

**品牌色（Primary Color）**
- 产品的主色调，体现品牌个性
- 用于主要按钮、高亮、选中状态
- 通常只有 1 种主色

**辅助色（Secondary Color）**
- 辅助主色，提供视觉变化
- 用于次要操作、标签、图表
- 通常 1-2 种

**语义色（Semantic Colors）**
```
成功（Success）：绿色系   #52C41A  → 操作成功、通过
警告（Warning）：橙色系   #FAAD14  → 警示信息
错误（Error）：  红色系   #FF4D4F  → 操作失败、危险
信息（Info）：   蓝色系   #1890FF  → 提示信息
```

**中性色（Neutral Colors）**
```
文字色：   #1A1A1A / #333333 / #666666 / #999999
背景色：   #FFFFFF / #F5F5F5 / #F0F0F0 / #E0E0E0
边框色：   #D9D9D9 / #E8E8E8
分割线：   #F0F0F0
```

### 颜色量表（Color Scale）

为每个主色生成完整的深浅系列：

```
主色：#4299E1（蓝色）

50:   #EBF5FF  ← 最浅，背景色
100:  #BEE3F8
200:  #90CDF4
300:  #63B3ED
400:  #4299E1  ← 基础色（Base）
500:  #3182CE
600:  #2B6CB0
700:  #2C5282
800:  #2A4365
900:  #1A365D  ← 最深
```

**使用规则**：
- 400（基础色）：主要按钮、强调色
- 50-100：背景色、浅色标签底色
- 600-700：深色文字、深色按钮
- 200-300：图表、插图填色

## 7.4 色彩的情绪与语义

不同颜色传递不同情感，选色要与产品调性匹配：

| 颜色 | 情绪语义 | 适用行业 | 代表产品 |
|------|---------|---------|---------|
| **红色** | 热情、紧迫、警告、食欲 | 餐饮、电商、直播 | 美团、抖音 |
| **橙色** | 活力、友好、创意、促销 | 电商、社交 | 哔哩哔哩(早期)、Firefox |
| **黄色** | 乐观、注意、快乐 | 儿童、金融（慎用） | 美团（亮黄） |
| **绿色** | 自然、成功、健康、财务 | 健康、金融、环保 | 微信、工商银行 |
| **蓝色** | 信任、专业、科技、冷静 | 科技、金融、医疗 | 支付宝、Facebook |
| **紫色** | 奢华、创意、神秘、高端 | 奢侈品、美妆、游戏 | 摩尔勇士、Yahoo |
| **粉色** | 浪漫、温柔、女性、甜美 | 美妆、女性产品 | Airbnb(旧Logo) |
| **黑色** | 高端、奢华、简约、力量 | 奢侈品、数码 | Apple、Nike |
| **白色** | 简洁、纯净、空间感 | 医疗、科技、极简 | Apple Store |

## 7.5 深色模式（Dark Mode）

深色模式不是简单地把白色变黑色，需要重新设计色彩方案。

### 深色模式的颜色规则

```
浅色模式                  深色模式
────────────              ────────────
背景：#FFFFFF             背景：#121212（不要纯黑）
卡片：#F5F5F5             卡片：#1E1E1E
文字：#1A1A1A             文字：#EBEBEB（不要纯白）
边框：#E0E0E0             边框：#2C2C2C
```

**深色模式注意事项**：
- 阴影在深色模式中不明显，改用边框/轻微发光效果
- 颜色饱和度在深色背景上更突出，主色可适当降低饱和度
- 避免大面积高饱和颜色（会产生光晕效果，刺眼）
- 图片/插图要有深色版本

### Figma 深色模式实现

使用 Color Styles + 变量（Variables）实现一键切换：

```
定义语义化颜色变量：
  bg-primary：浅色=#FFFFFF，深色=#121212
  text-primary：浅色=#1A1A1A，深色=#EBEBEB
  surface：浅色=#F5F5F5，深色=#1E1E1E

切换时：修改变量集的 Mode，所有使用该变量的地方自动更新
```

## 7.6 色彩无障碍

### 对比度要求（WCAG 2.1）

```
AA 级别（最低要求）：
  普通文字（< 18px）：对比度 ≥ 4.5:1
  大文字（≥ 18px，或 ≥ 14px Bold）：对比度 ≥ 3:1

AAA 级别（更高要求）：
  普通文字：对比度 ≥ 7:1
  大文字：对比度 ≥ 4.5:1
```

**计算对比度**：
```
对比度 = (L1 + 0.05) / (L2 + 0.05)

其中 L1 是较亮颜色的相对亮度，L2 是较暗颜色的相对亮度

工具：使用 Figma 插件 "A11y - Color Contrast Checker"
```

### 色盲友好设计

全球约 8% 的男性和 0.5% 的女性有色觉缺陷，最常见的是红绿色盲。

**红绿色盲看到的颜色**：
```
正常人看到：绿色成功，红色错误
色盲者看到：两种颜色可能相似

解决方案：不要仅靠颜色传递信息
  ✗ 错误：表格用红/绿背景区分好坏
  ✓ 正确：在颜色基础上添加图标（✓/✗）或文字标签
```

**工具**：Figma 插件 "Color Blind" 可模拟色盲视角

## 7.7 实战：为产品设计配色方案

### 步骤一：确定品牌调性
```
问题清单：
• 产品面向什么用户？（年龄、性别、职业）
• 产品想传递什么情感？（信任/活力/专业/亲切）
• 竞品用什么颜色？（差异化还是跟随？）
• 使用场景是什么？（白天/夜晚/户外）
```

### 步骤二：选择主色
```
1. 根据调性找参考色（Pinterest、Dribbble 找灵感）
2. 在 Figma 或 HSB 调色中微调到合适的主色
3. 验证主色在白底上的对比度（≥ 4.5:1）
4. 验证白色文字在主色上的对比度（用于按钮内文字）
```

### 步骤三：生成颜色量表
```
工具推荐：
• Tailwind CSS Color Generator
• Huemint.com（AI 配色）
• Coolors.co（配色方案生成）
• ColorBox.io（量表生成）
```

### 步骤四：定义中性色
```
从主色中提取色相，混入中性色系，避免使用纯灰色：

纯灰：#666666（与主色脱节，显得廉价）
带色相的灰：如蓝调灰 #5A6B7A（与蓝色主色呼应，更高级）

方法：在 HSB 中保留 5-10% 的主色色相，降低饱和度
```

## 本章小结

| 概念 | 要点 |
|------|------|
| HSB 模型 | 色相/饱和度/明度，最适合 UI 调色 |
| 配色方案 | 单色/邻近色/互补色，根据产品调性选择 |
| 色彩角色 | 主色/辅助色/语义色/中性色四类 |
| 情绪语义 | 不同颜色传递不同情感，要与品牌匹配 |
| 无障碍 | 对比度 ≥ 4.5:1，不单独依赖颜色传递信息 |
