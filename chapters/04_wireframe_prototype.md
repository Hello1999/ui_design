# 第 04 章：线框图与原型

> 目标：掌握从草图到高保真原型的完整流程，用低成本方式快速验证设计想法。

---

## 4.1 原型的保真度层次

**保真度（Fidelity）** 指原型与最终产品的相似程度。

```
低保真                    中保真                    高保真
────────────────────────────────────────────────────────→
纸质草图              数字线框图                像素级还原
手绘方框              灰度占位符                真实颜色/字体
文字标注              可点击跳转                完整交互动效
──────────            ──────────                ──────────
目的：快速探索         目的：验证流程             目的：视觉验收
时间：分钟级           时间：小时级               时间：天级
受众：内部团队         受众：内部+利益相关方        受众：用户测试
```

**关键原则**：在正确的阶段使用正确的保真度。过早做高保真是时间浪费。

## 4.2 手绘草图

草图是最快的原型方式，成本极低，修改无压力。

### 草图技巧

**基本图形语言**：
```
┌─────┐   矩形 = 内容区域/卡片
└─────┘

[  按钮  ]   方括号 = 按钮

───────   横线 = 文字内容（用线代替真实文字）

╔═══════╗   双线框 = 图片/图像区域
║  img  ║
╚═══════╝

⬡  图标用简单几何形状代替

≡  三条横线 = 列表/菜单
```

**手机屏幕模板**：
```
  ┌─────────────┐
  │ 9:41     ▓▓▓│  ← 状态栏
  ├─────────────┤
  │  ←  标题    │  ← 导航栏
  ├─────────────┤
  │             │
  │  内容区域   │
  │             │
  │             │
  ├─────────────┤
  │ ⊙  ⊙  ⊙  ⊙ │  ← 标签栏
  └─────────────┘
```

### 快速草图的 Crazy 8s 方法

1. 将 A4 纸折叠成 8 格
2. 8 分钟内，每格画一个不同的界面方案
3. 不允许超时，强迫快速出方案
4. 选出最有潜力的 1-2 个方案深化

## 4.3 线框图（Wireframe）

线框图是数字化的低保真原型，关注布局和功能，不关注视觉美感。

### 线框图的设计原则

**1. 灰度原则**：只用黑、白、灰，避免颜色分散注意力
**2. 内容优先**：用真实（或接近真实）的文案，避免乱码占位符
**3. 标注清晰**：关键交互逻辑用注释说明
**4. 聚焦结构**：只表达"有什么"和"在哪里"，不表达"怎么好看"

### Figma 线框图实践

**页面元素命名规范**：
```
框架命名：[功能名]-[页面名]
例：cart-checkout（购物车-结算页）

图层命名：[类型]/[描述]
例：button/primary-confirm
     text/title-h1
     image/product-thumbnail
```

**常用线框图组件**：
```
导航栏：
┌─────────────────────────────────┐
│  ←  [页面标题]              ···  │
└─────────────────────────────────┘

卡片：
┌─────────────────────────────────┐
│ ████  标题文字                   │
│ img   副标题说明文字              │
│       ──────  按钮               │
└─────────────────────────────────┘

表单项：
  标签
  ┌─────────────────────────────┐
  │ 请输入内容                   │
  └─────────────────────────────┘
  提示说明文字

```

## 4.4 交互流程（Interaction Flow）

交互流程将多个线框图页面连接起来，展示完整的用户操作路径。

### 流程图示例：登录流程

```
[启动页]
    │
    ├──（已有账号）→ [登录页] → （输入账号密码）
    │                              │
    │                      ┌───────┴───────┐
    │                      ↓               ↓
    │              [密码错误提示]      [登录成功→首页]
    │
    └──（没有账号）→ [注册页] → [完善信息] → [首页]
```

### 在 Figma 中创建交互流程

1. 创建所有页面的线框图
2. 使用 **Prototype** 模式连接页面
3. 设置触发条件：On Tap / On Click / On Hover
4. 设置动画类型：Push / Slide / Dissolve
5. 预览并测试流程是否顺畅

## 4.5 高保真原型（High-Fidelity Prototype）

高保真原型包含真实的视觉设计，用于用户测试和开发交付。

### 高保真原型的构成要素

| 要素 | 说明 |
|------|------|
| 真实颜色 | 使用品牌色彩方案 |
| 真实字体 | 使用目标字体和字号 |
| 真实图片 | 使用真实产品图或接近真实的占位图 |
| 真实文案 | 使用真实的界面文字 |
| 组件规范 | 使用统一的按钮、输入框等组件 |
| 可交互 | 点击跳转、滑动、弹窗等交互效果 |

## 4.6 Figma 核心操作指南

### 界面结构

```
┌──────────┬─────────────────────────────┬──────────────┐
│          │                             │              │
│  图层面板 │         画布区域             │  属性面板     │
│  Layers  │                             │  Properties  │
│          │  Pages:                     │              │
│  ▸ Page1 │  [Frame] [Frame] [Frame]    │  W: 390      │
│  ▸ Page2 │                             │  H: 844      │
│          │                             │  X: 0  Y: 0  │
└──────────┴─────────────────────────────┴──────────────┘
```

### 常用快捷键（必记）

| 操作 | Mac | Windows |
|------|-----|---------|
| 创建框架（Frame） | F | F |
| 矩形 | R | R |
| 文字 | T | T |
| 选择 | V | V |
| 缩放到全图 | Shift+1 | Shift+1 |
| 缩放到选中 | Shift+2 | Shift+2 |
| 组合 | Cmd+G | Ctrl+G |
| 创建组件 | Cmd+Alt+K | Ctrl+Alt+K |
| 复制样式 | Cmd+Alt+C | Ctrl+Alt+C |
| 粘贴样式 | Cmd+Alt+V | Ctrl+Alt+V |
| 自动布局 | Shift+A | Shift+A |

### Auto Layout（自动布局）

Auto Layout 是 Figma 最重要的功能之一，实现响应式布局：

```
无 Auto Layout：             有 Auto Layout：
┌────────────┐               ┌──────────────────────┐
│ [按钮文字]  │  文字变长时    │ [按钮文字变长了也OK]  │
└────────────┘  框不会自动   └──────────────────────┘
                扩展          文字变长，框自动扩展
```

**设置方式**：选中元素 → 右键 → Add Auto Layout（或 Shift+A）

**关键属性**：
- Direction：水平/垂直排列
- Gap：元素间距
- Padding：内边距
- Fill/Fixed：是否自适应宽高

### 组件（Components）

组件是可复用的 UI 元素，修改主组件会同步更新所有实例：

```
创建组件：选中元素 → Cmd+Alt+K（创建主组件）
使用组件：从资源面板拖入，或复制主组件（创建实例）
覆盖属性：实例可独立修改颜色、文字等，不影响其他实例
```

**Variants（变体）** 可将同一组件的不同状态合并：
```
Button 组件变体：
├── Type: Primary / Secondary / Destructive
├── Size: Large / Medium / Small
└── State: Default / Hover / Pressed / Disabled
```

## 4.7 原型工具对比

| 工具 | 特点 | 适用场景 |
|------|------|---------|
| **Figma** | 协作、组件化、Web端 | 主流推荐，UI/UX 全流程 |
| **Axure** | 逻辑交互强大 | 复杂交互、条件判断 |
| **ProtoPie** | 动效逼真 | 演示高保真交互动效 |

| **Marvel** | 简单易用 | 快速原型验证 |
| **Framer** | 代码级交互 | 开发者友好 |

## 4.8 原型测试前的检查清单

在给用户或开发团队展示原型前，检查以下项目：

**流程完整性**
- [ ] 所有关键路径都可以走通
- [ ] 错误状态（空状态、加载状态）都有设计
- [ ] 返回/取消操作都有处理

**视觉一致性**
- [ ] 同类元素样式统一
- [ ] 间距使用统一的规范值
- [ ] 字体层级清晰一致

**交互合理性**
- [ ] 点击区域足够大（至少 44px×44px）
- [ ] 关键操作有反馈提示
- [ ] 加载过程有状态提示

## 本章实践作业

**作业 1**：用 Figma 绘制一个**3 页面线框图**：
- 主页（内容列表）
- 详情页（内容展示）
- 个人中心（账户信息）

要求：灰度配色，包含真实文案，标注关键交互

**作业 2**：将三个页面用 Prototype 连接，实现：
- 从列表点击进入详情页
- 详情页可以返回列表页
- 导航栏可以切换到个人中心

**作业 3**：把原型分享给 1 位朋友，观察他们如何操作（不要引导），记录卡顿的地方。

## 本章小结

| 概念 | 要点 |
|------|------|
| 保真度层次 | 低保真草图 → 中保真线框 → 高保真视觉稿 |
| 线框图原则 | 灰度、内容优先、聚焦结构 |
| Figma 核心 | Auto Layout、组件、Prototype 模式 |
| 原型目的 | 低成本验证，发现问题，而非追求完美 |
