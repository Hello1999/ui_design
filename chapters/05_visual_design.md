# 第 05 章：视觉设计基础

> 目标：掌握视觉设计的底层原理，理解为什么"好看"的设计背后有规律可循。

---

## 5.1 格式塔原理（Gestalt Principles）

格式塔是德文"整体"的意思，格式塔心理学研究人类如何感知视觉信息。设计师必须了解这些原理。

### 六大格式塔原理

**1. 亲密性（Proximity）**
> 距离近的元素被视为一组

```
●  ●  ●        ●●●    ○○○
●  ●  ●   vs   ●●●    ○○○
●  ●  ●        ●●●    ○○○

左图：看起来是一个9个点的整体
右图：看起来是两组，各3×3
```

**实践**：相关元素（标签和输入框）之间的间距要小于不相关元素之间的间距。

**2. 相似性（Similarity）**
> 形状、颜色、大小相似的元素被视为一组

```
●  ●  ■  ●  ●
●  ●  ■  ●  ●

■ 的一列视觉上独立出来，即使位置相邻
```

**实践**：同类按钮使用相同颜色，主操作和次操作用不同样式区分。

**3. 连续性（Continuity）**
> 人眼倾向于沿着线条/曲线的方向延伸

```
─────────●─────────

人眼会把两段线段感知为一条穿越圆点的线，而非两条分开的线
```

**实践**：列表、进度条、时间轴都利用了连续性原理。

**4. 封闭性（Closure）**
> 人脑会自动补全不完整的形状

```
缺了角的矩形仍然被识别为矩形：
┌───
│
└───     ←  右侧没有线段，但你仍看到矩形
```

**实践**：图标设计可以利用封闭性，减少细节同时保持可识别性。

**5. 主体/背景（Figure/Ground）**
> 人眼总是区分"主要内容"和"背景"

```
白底黑字 vs 卡片浮于背景之上
→ 利用对比度和阴影强调主体内容
```

**6. 对称性（Symmetry）**
> 对称的元素被视为有关联

**实践**：模态弹窗的左右按钮应保持对称，强调选择的平衡性。

## 5.2 CRAP 设计原则

CRAP 是四个视觉设计基本原则的缩写，由 Robin Williams 在《写给大家看的设计书》中提出。

### C - 对比（Contrast）

> 如果两个元素不同，就要让它们**明显不同**。

**避免**：相似但不同的元素（模糊的层级关系）

```
✗ 错误：
  标题文字（16px 灰色）
  正文文字（14px 深灰色）
  ← 差别太小，层级不清晰

✓ 正确：
  标题文字（24px 黑色 Bold）
  正文文字（14px 中灰色 Regular）
  ← 大小、颜色、粗细形成强烈对比
```

**对比的维度**：
- 大小对比（大标题 vs 小正文）
- 颜色对比（深色 vs 浅色）
- 粗细对比（Bold vs Light）
- 形状对比（圆角 vs 直角）
- 纹理对比（有纹理 vs 纯色）

### R - 重复（Repetition）

> 设计中的某些元素要在整个作品中**重复出现**。

**目的**：建立视觉一致性，让用户感知到"这些东西属于同一个系统"

```
重复的元素可以是：
• 颜色：主色调蓝色在所有可点击元素上出现
• 形状：所有卡片使用相同圆角半径（8px）
• 间距：所有内边距使用 16px 或其倍数
• 字体：标题统一用 SF Pro，正文统一用系统字体
```

**实践**：这就是**设计系统**的核心价值——重复的规范。

### A - 对齐（Alignment）

> 每个元素都应该与页面上的某个其他元素有**视觉连接**。

**不要随意放置元素**：
```
✗ 错误（随机对齐）：
  标题
      副标题（缩进不明原因）
  ─────按钮（居中）
  正文文字（左对齐）

✓ 正确（左对齐基准线）：
  标题
  副标题
  正文文字
  [按钮]
  ← 所有元素共享同一条左基准线
```

**对齐类型**：
- 左对齐：最自然，适合大段文字
- 居中对齐：适合标题、卡片内容
- 右对齐：适合数字、价格
- 两端对齐：适合宽幅正文（慎用，中文排版有时会有字距问题）

### P - 亲密性（Proximity）

（与格式塔亲密性原则相同，此处强调设计实践）

> 相关的信息要**物理上靠近**，不相关的信息要有足够的间距分隔。

```
✗ 错误：
  头像    姓名
  评分    关注按钮

  所有元素间距相同，不知道哪些属于同一组

✓ 正确：
  [头像 + 姓名 + 评分]    [关注按钮]

  头像/姓名/评分是一组（用户信息），
  关注按钮与用户信息有明显间距
```

## 5.3 视觉层级（Visual Hierarchy）

视觉层级控制用户注意力的先后顺序，引导用户从最重要的信息看到次要信息。

### 建立视觉层级的工具

| 工具 | 效果 |
|------|------|
| **大小** | 越大越重要 |
| **粗细** | 越粗越重要 |
| **颜色** | 高饱和度、高对比度 = 重要 |
| **位置** | F型/Z型视觉动线的起点最重要 |
| **空白** | 被大量空白包围的元素更突出 |
| **阴影/叠加** | 浮在上层的元素更重要 |

### 界面层级示例

```
一级层级（最重要）：页面标题、主操作按钮
  ↓
二级层级（重要）：内容标题、关键数据
  ↓
三级层级（次要）：描述文字、辅助信息
  ↓
四级层级（低调）：时间戳、底部声明文字
```

**实践模板**（新闻卡片）：
```
┌────────────────────────────┐
│ ████ 大标题（Bold，18px）    │  ← 一级
│ ████ ████（Bold，18px）     │
│                            │
│ 副标题概括（Regular，14px）  │  ← 二级
│ 副标题内容在这里             │
│                            │
│ 来源  •  时间（12px 灰色）   │  ← 三级
└────────────────────────────┘
```

## 5.4 空白（White Space/Negative Space）

空白不是浪费，而是设计的重要元素。

### 空白的作用

- **呼吸感**：内容之间有空隙，不显拥挤
- **聚焦**：大量空白包围重要内容，突出重点
- **层级**：组与组之间用较大空白区分

### 空白的两种类型

```
微空白（Micro White Space）：
  元素内部的空间
  如：按钮的内边距、行间距、字间距

宏空白（Macro White Space）：
  区块之间、页面边缘的空间
  如：卡片之间的间距、顶部安全区
```

### 间距规范

推荐使用 **8pt 栅格系统**：所有间距都是 8 的倍数

```
4px  - 极小间距（图标与文字之间）
8px  - 小间距（行内元素间距）
12px - 中小间距
16px - 基础间距（标准内边距）
24px - 中间距（组内元素间距）
32px - 大间距（不同内容区块间距）
48px - 更大间距（章节级分隔）
64px - 超大间距（页面级分隔）
```

## 5.5 视觉动线

用户阅读屏幕时，眼睛会按照一定规律移动。

### F 型视觉动线

研究表明，用户阅读网页时的眼动轨迹呈 F 型：

```
→→→→→→→→→→→→→→→→→→→  （第一行，完整阅读）
→→→→→→→→→→→→→→→→      （第二行，次完整阅读）
→→→→                    （左侧快速扫描）
→→
```

**设计启示**：
- 最重要的信息放在左上角和第一行
- 导航和关键操作放在顶部
- 列表内容的左侧第一个词最重要

### Z 型视觉动线

营销页/落地页常见的阅读模式：

```
→→→→→→→→→
           ↘
            →→→→→→→
                    ↘
→→→→→→→→→
           ↘
            →→→→→→→
```

**设计启示**：
- Logo 放左上，CTA（行动按钮）放右侧
- 核心内容沿 Z 型路径排列

## 5.6 视觉设计常见错误

| 错误 | 表现 | 解决方案 |
|------|------|---------|
| 元素样式太相似 | 所有文字看起来差不多 | 增大差异，加强对比 |
| 元素间距不一致 | 同类元素有时 8px 有时 12px | 建立间距规范，严格执行 |
| 对齐混乱 | 元素位置随意 | 使用网格和基准线 |
| 过度使用色彩 | 页面有 7 种颜色 | 主色+辅色+中性色，控制在 3-4 种 |
| 文字对比度不足 | 浅灰文字在白底上 | 确保文字对比度 ≥ 4.5:1 |

## 本章小结

| 原则 | 核心要点 |
|------|---------|
| 格式塔 | 亲密性、相似性、连续性、封闭性影响感知 |
| CRAP | 对比、重复、对齐、亲密性四大设计原则 |
| 视觉层级 | 用大小、粗细、颜色引导用户注意力 |
| 空白 | 空白是设计元素，不是浪费 |
| 视觉动线 | F型和Z型动线，最重要内容放在起点 |
