# 第 06 章：字体排版

> 目标：掌握字体选择和排版规范，让文字传递信息的同时也具备美感和可读性。

---

## 6.1 字体基础知识

### 字体解剖

```
              上延伸线 (Ascender)
              │
   ┌──────────│─────────────────────────────┐
   │          │    大写高度                  │ ← 大写字母顶部
   │  H       │    (Cap Height)             │
   │          │                             │
基线─│──────────│─────────────────────────────│──── Baseline
   │          │                             │
   │          │    下延伸线 (Descender)      │
   └──────────│─────────────────────────────┘
              │
              p, g, y 等字母下沉到这里
```

**重要概念**：
- **x-height**：小写字母 x 的高度，影响可读性（x-height 越大越易读）
- **Baseline**：基准线，所有字符底部对齐的参考线
- **Leading（行距）**：两行基准线之间的距离
- **Tracking（字间距）**：字母之间的整体间距
- **Kerning（字偶距）**：特定字母对之间的间距调整

## 6.2 字体分类

### 西文字体

| 类型 | 特征 | 氛围 | 适用场景 |
|------|------|------|---------|
| **衬线体（Serif）** | 笔画末端有衬线 | 传统、权威、信任 | 正文、新闻、书籍 |
| **无衬线体（Sans-serif）** | 笔画干净无装饰 | 现代、简洁、友好 | 屏幕 UI、科技产品 |
| **等宽体（Monospace）** | 每个字符宽度相同 | 技术、代码 | 代码展示、终端 |
| **手写体（Script）** | 模拟手写效果 | 个性、情感、创意 | 标志、装饰性标题 |
| **展示体（Display）** | 装饰性强 | 独特、引人注目 | 大标题、海报 |

```
衬线体示例：Georgia, Times New Roman, Garamond
           I have serifs on my letters

无衬线体示例：SF Pro, Inter, Roboto, Helvetica
             I have no serifs
```

### 中文字体

| 类型 | 代表字体 | 适用场景 |
|------|---------|---------|
| **黑体（Gothic）** | 苹方、思源黑体、微软雅黑 | 屏幕 UI、标题、正文 |
| **宋体（Ming）** | 宋体、思源宋体 | 印刷正文、传统感 |
| **圆体** | 华文圆体 | 亲切、年轻化产品 |
| **楷体** | 楷体 | 传统文化、书法感 |

**屏幕 UI 推荐**：
- 中文：苹方（iOS）、思源黑体（跨平台）、微软雅黑（Windows）
- 英文：SF Pro（iOS/macOS）、Roboto（Android）、Inter（Web）

## 6.3 字体层级系统

UI 设计通常需要 4-6 个字体层级，覆盖从标题到辅助信息的所有文字。

### 移动端字体层级（iOS 规范参考）

| 层级 | 名称 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|------|
| H1 | Large Title | 34px | Bold(700) | 41px | 页面大标题 |
| H2 | Title 1 | 28px | Bold(700) | 34px | 一级标题 |
| H3 | Title 2 | 22px | Bold(700) | 28px | 二级标题 |
| H4 | Headline | 17px | Semibold(600) | 22px | 列表标题 |
| Body | Body | 17px | Regular(400) | 22px | 正文内容 |
| Sub | Subheadline | 15px | Regular(400) | 20px | 副标题 |
| Caption | Caption | 12px | Regular(400) | 16px | 辅助说明 |

### 中文字体层级建议

```
页面标题：   24-28px  Bold    #1A1A1A（近黑色）
一级标题：   20px     Bold    #1A1A1A
二级标题：   17px     Medium  #1A1A1A
正文：       15px     Regular #333333
辅助信息：   13px     Regular #666666
标签/角标：  11px     Regular #999999
```

**原则**：
- 正文最小不低于 12px（移动端建议 13px+）
- 同屏最多使用 3 种字号
- 通过**粗细**而非仅靠**字号**建立层级

## 6.4 中文排版规范

### 字间距（字符间距）

```
标题：        可适当增加字间距（0.5-2px），让标题更舒展
正文：        使用默认字间距（0），过大的字间距影响连贯性
全大写英文：  建议增加字间距（让大写字母不显拥挤）
```

### 行距（Line Height）

```
行距 = 字号 × 倍数

标题（单行/两行）：    1.2 - 1.3 倍  （紧凑，强调整体感）
副标题（2-3行）：      1.4 - 1.5 倍
正文（多行）：          1.6 - 1.8 倍  （宽松，易于阅读）
辅助说明（多行）：      1.5 - 1.6 倍
```

**示例**：
```
14px 正文 × 1.6 = 22.4px 行高（取整为 22px）
```

### 行长（每行字符数）

```
最佳阅读行长：中文 25-40 字/行，英文 45-75 字/行

太短（< 20 字）：眼睛跳行太频繁，阅读疲劳
太长（> 50 字）：难以找到下一行的起点

移动端：320px 宽 × 15px 字 ≈ 约 16-20 字（合适）
```

### 标点符号规范

| 规则 | 说明 |
|------|------|
| 中文使用全角标点 | 逗号，句号。问号？ |
| 中英文混排时的空格 | 中文与英文/数字之间加空格：我用 iPhone 15 |
| 不用连续空格 | 用 padding/margin 控制间距 |
| 不在行首出现标点 | 避免句号、逗号出现在行首 |

## 6.5 字体搭配

### 原则

1. **同族搭配**：使用同一字体家族的不同字重（最安全）
2. **对比搭配**：衬线 + 无衬线（传统感 + 现代感的结合）
3. **最多两种字体**：超过两种字体显得杂乱

### 经典搭配组合

**中文界面（推荐）**：
```
标题：苹方-中 / 思源黑体 Bold
正文：苹方-细 / 思源黑体 Regular
英文和数字：SF Pro / Inter（作为中文字体的补充）
```

**英文界面**：
```
搭配 1：Inter（无衬线）+ Playfair Display（衬线装饰标题）
搭配 2：全用 Inter，靠字重建立层级（最常见）
搭配 3：SF Pro（苹果生态系统）
搭配 4：Roboto（Google Material 生态）
```

## 6.6 文字颜色与可读性

### 颜色对比度

Web 内容可访问性指南（WCAG 2.1）规定：
```
普通文字（14px 以下）：对比度 ≥ 4.5:1
大文字（18px+，或 14px+ Bold）：对比度 ≥ 3:1
```

### 常用文字颜色方案

```
在白色背景（#FFFFFF）上：

#000000  对比度 21:1  ← 最强，适合标题
#1A1A1A  对比度 16:1  ← 柔和的黑，适合标题/正文
#333333  对比度 12:1  ← 深灰，适合正文
#666666  对比度  5.7:1 ← 中灰，适合辅助信息（通过AA）
#999999  对比度  2.85:1 ← 浅灰，⚠️ 低于标准，仅用于14px以上大字
#BBBBBB  对比度  1.73:1 ← ✗ 不符合任何可读性标准，禁止用于文字
```

**工具**：Figma 的 Contrast Checker 插件，或 [Contrast Ratio 网站]

## 6.7 数字排版

数字在界面中频繁出现（价格、统计、评分），需要特别注意：

```
等宽数字（Tabular Figures）：
  不使用：1  2  3  ← 字宽不同，列表对不齐
  推荐：  1  2  3  ← 字宽相同，数字列整齐

金额展示：
  ✓  ¥ 1,234.56  （千分位分隔符）
  ✗  ¥1234.56    （难以快速读取大数字）

价格对比：
  ✓  ¥299  ~~¥599~~  （划线价与现价对比鲜明）
```

## 6.8 字体加载与性能

Web 端注意事项：

```css
/* 字体回退栈（Font Stack）推荐 */
/* 中文网站 */
font-family: "PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;

/* 英文网站 */
font-family: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

**系统字体优势**：加载速度快，与系统视觉统一，适合大多数 App

**自定义字体注意**：
- 只加载需要的字重（不要加载全部 9 个字重）
- 使用 `font-display: swap` 避免闪烁
- 中文字体文件通常很大（5-30MB），使用子集化
