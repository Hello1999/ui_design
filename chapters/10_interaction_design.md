# 第 10 章：交互设计与微交互

> 目标：掌握交互设计的核心原则，理解如何通过微交互提升用户体验的细腻感。

---

## 10.1 交互设计基础

### 什么是交互设计

交互设计（Interaction Design，IxD）关注的是**用户与产品之间的对话**：
- 用户发起操作（输入）
- 系统作出响应（反馈）
- 循环往复，形成交互

### 交互的五个维度

Bill Moggridge 和 Gillian Crampton Smith 提出交互设计的五个维度：

| 维度 | 描述 | 示例 |
|------|------|------|
| **文字（Words）** | 标签、按钮文案、提示语 | "确认" vs "知道了" |
| **视觉呈现（Visuals）** | 图像、图标、动效 | 加载动画、错误图示 |
| **物理空间（Space）** | 布局、间距、手势区域 | 底部操作区、滑动区域 |
| **时间（Time）** | 动画时长、延迟、节奏 | 250ms 过渡动画 |
| **行为（Behavior）** | 输入和输出的逻辑 | 点击按钮后发生了什么 |

## 10.2 交互原则

### 1. 可见性（Visibility）

用户需要的功能应该是可见的或可发现的。

```
✗ 错误：关键功能藏在长按菜单里，用户不知道
✓ 正确：主要操作在界面上直接可见

可见性等级：
  直接可见  →  点一下可见  →  需要探索才能发现
  （最佳）                       （最差）
```

### 2. 反馈（Feedback）

每个用户操作都应该有明确的反馈。

```
操作 → 反馈 的时间要求：

0-100ms：  感觉是即时的（触摸、点击）
100-300ms：感觉流畅（动画过渡）
300-1000ms：用户能感知到延迟，需要视觉提示
> 1000ms：必须显示加载状态（进度条/Spinner）
> 10秒：需要提供取消选项
```

**反馈类型**：
```
视觉反馈：按钮按下时变色、加载动画
听觉反馈：消息提示音、点击音效（慎用）
触觉反馈：震动反馈（Haptic，移动端）
信息反馈：Toast 提示、成功页面
```

### 3. 约束（Constraints）

通过设计限制用户的错误行为。

```
示例：
• 日期选择器：限制用户只能选择有效日期
• 手机号输入框：只允许输入数字
• 表单提交：填写完成前禁用提交按钮
• 危险操作：需要二次确认（"确定要删除吗？"）
```

### 4. 一致性（Consistency）

相同的操作产生相同的结果，相似的元素有相似的行为。

```
内部一致性：
  所有页面的返回按钮都在左上角
  所有弹窗都从底部弹出
  所有删除操作都需要确认

外部一致性（平台惯例）：
  iOS：左滑删除列表项
  Android：返回按钮在导航栏
  Web：Logo 点击回首页
```

### 5. 容错（Error Recovery）

允许用户犯错，并提供简单的恢复路径。

```
撤销（Undo）：误操作后可以撤销
  Gmail 发送邮件后的 5 秒撤回窗口

确认（Confirm）：不可逆操作前二次确认
  "删除后无法恢复，确认删除？"

软删除（Soft Delete）：数据不立即清除，进回收站
  文件移入废纸篓，30天后才真正删除
```

## 10.3 状态设计

界面中的每个元素都有多种状态，必须全部设计。

### 页面级状态

```
1. 加载中（Loading）
   ─────────────────
   [Skeleton 骨架屏]  或  [Spinner]

2. 空状态（Empty State）
   ─────────────────
   [插图]
   暂无内容
   [去创建第一个]

3. 错误状态（Error State）
   ─────────────────
   [错误图示]
   加载失败
   [重试]

4. 成功状态（Success State）
   ─────────────────
   ✓ 操作成功！

5. 正常状态（Default）
   正常内容展示
```

### 元素级状态

```
按钮状态：
Default → Hover → Active(Pressed) → Focus → Disabled → Loading

输入框状态：
Default → Focus → Filled → Error → Success → Disabled

列表项状态：
Default → Hover → Selected → Disabled

切换（Toggle）：
Off → On → Disabled-Off → Disabled-On
```

### Skeleton Screen（骨架屏）

骨架屏是比 Spinner 更优的加载状态：

```
Spinner（差）：           骨架屏（好）：
                          ┌──────────────────────┐
                          │ ████  ──────────      │
  ⟳                       │ ████  ──────────      │
  加载中                   │ ████  ──────────      │
                          └──────────────────────┘
  用户不知道等多久           用户预期了内容形态，感知时间更短
```

## 10.4 微交互（Micro-interactions）

微交互是精心设计的小型交互时刻，提升产品的精致感和人性化。

### 微交互的四个部分（Dan Saffer 模型）

```
触发器（Trigger）  →  规则（Rules）  →  反馈（Feedback）  →  循环/模式（Loops）

用户摇一摇         触发摇一摇功能      手机震动            每次摇一摇都重复
```

### 经典微交互案例

**1. 下拉刷新**
```
用户操作：下拉屏幕
动画反馈：出现刷新图标，随下拉距离旋转
触发规则：下拉超过阈值松手后触发刷新
完成反馈：图标消失，内容更新
```

**2. 点赞动效**
```
用户操作：双击图片 或 点击爱心图标
动效反馈：
  • 爱心图标从实心变空心（或反之）
  • 缩放动画：从 1.0 → 1.3 → 1.0
  • 粒子爆炸效果（Instagram 双击点赞）
  • 数字跳动：+1
```

**3. 表单验证**
```
输入邮箱格式错误时：
  • 输入框边框变红
  • 轻微抖动动画（震动暗示"错了"）
  • 显示错误提示文字
```

**4. 购物车添加**
```
点击"加入购物车"：
  • 按钮出现 Loading 状态（防重复点击）
  • 商品图片飞向购物车图标（飞入动效）
  • 购物车数字徽标 +1，带弹跳动画
  • 按钮恢复正常，文字变"已加入"
```

**5. 密码强度**
```
用户输入密码时实时显示：
弱 ██░░░░  （红色）
中 ████░░  （橙色）
强 ██████  （绿色）
```

## 10.5 动效设计原则

### 12 条动画基本原则（Disney 动画师总结）

UI 动效最常用的几条：

**1. 挤压与拉伸（Squash and Stretch）**
```
模拟真实物体的弹性，增加活力
示例：按钮按下时轻微压扁
```

**2. 预备动作（Anticipation）**
```
主动作之前有微小的反向动作
示例：列表项删除前先轻微向右偏移
```

**3. 跟随与重叠（Follow Through & Overlapping）**
```
主体停止后，附属部分还在继续运动
示例：弹窗出现后，内容有轻微的继续往下的惯性
```

**4. 缓动（Easing）**
```
物体运动不是匀速的，而是有加速和减速

ease-in：慢→快（元素离开屏幕）
ease-out：快→慢（元素进入屏幕）
ease-in-out：慢→快→慢（元素在屏幕内移动）
```

### UI 动效时长规范

```
微小动效（图标变化、颜色过渡）：  100-150ms
元素进入/离开：                  200-300ms
页面转场：                        300-500ms
复杂动效（引导、空状态）：         500-1000ms

经验法则：移动设备用户体验的最佳动效时长：200-300ms
```

### Easing 曲线

```
线性（Linear）：   ──────  匀速，机械感，避免使用
ease-out：        ⌒─────  由快到慢，元素进入（最常用）
ease-in：         ─────⌣  由慢到快，元素离开
ease-in-out：     ⌒──⌣    完整弧线，界面内移动
spring（弹性）：   ⌒⌣⌒⌣   有弹跳感，用于有趣的交互
```

## 10.6 手势交互

移动端的特有交互方式：

### 常见手势

| 手势 | 操作 | 常见用途 |
|------|------|---------|
| 点击（Tap） | 单指轻触 | 选择、确认、打开 |
| 双击（Double Tap） | 快速点击两次 | 缩放、点赞 |
| 长按（Long Press） | 持续按住 > 0.5s | 进入编辑模式、上下文菜单 |
| 左滑（Swipe Left） | 向左滑动 | 删除、归档（列表操作） |
| 右滑（Swipe Right） | 向右滑动 | 完成、返回上一页 |
| 上滑（Swipe Up） | 从底部上滑 | 关闭弹窗、返回主界面 |
| 下拉（Pull Down） | 从顶部下拉 | 刷新内容 |
| 捏合/展开（Pinch） | 两指相对/相背 | 缩放图片/地图 |
| 旋转（Rotate） | 两指旋转 | 旋转图片/地图 |

### 手势设计原则

```
1. 提供视觉线索：让用户知道哪里可以滑动
   示例：列表项轻微透出删除按钮，暗示可以左滑

2. 渐进式手势：操作随手势进度实时反馈
   示例：下拉刷新随下拉距离显示进度

3. 允许取消：手势可以拖回起点取消
   示例：左滑超过阈值自动触发删除，未超过则弹回

4. 不要依赖手势作为唯一入口
   重要功能必须有直接可见的按钮，手势是附加操作
```

## 10.7 Figma 原型交互

在 Figma 中实现交互效果：

### 基础交互设置

```
连接方式：
  On Click       → 点击后触发（最常用）
  While Pressing → 按住时触发
  On Drag        → 拖拽触发
  On Hover       → 悬停触发（Web端）
  Mouse Enter/Leave → 进入/离开触发

动画类型：
  Instant       → 无动画，立即切换
  Dissolve      → 淡入淡出
  Smart Animate → 智能动画（需相同图层名，实现补间动画）
  Push          → 推入（适合页面导航）
  Slide In      → 滑入（适合弹窗）
  Move In       → 移入

时长：建议 200-300ms
曲线：建议 Ease Out
```

### Smart Animate 使用方法

1. 两帧使用**相同图层名称**的元素
2. 连接两帧时选择 **Smart Animate**
3. Figma 自动计算元素变化，生成补间动画

```
示例：按钮 Loading 状态

Frame 1（Default）：
  - 图层"button-text" = "提交"
  - 图层"spinner"（隐藏）

Frame 2（Loading）：
  - 图层"button-text" = "提交中..." （颜色变浅）
  - 图层"spinner"（显示，旋转45°）

Smart Animate 会自动动画过渡文字颜色和 spinner 的出现
```

## 本章小结

| 概念 | 要点 |
|------|------|
| 交互原则 | 可见性、反馈、约束、一致性、容错 |
| 状态设计 | 加载/空/错误/成功，每个状态都要设计 |
| 微交互 | 触发→规则→反馈→循环，小细节大体验 |
| 动效原则 | Ease Out 进入，200-300ms 最佳时长 |
| 手势交互 | 提供视觉线索，允许取消，不单一依赖手势 |
