# 第 14 章：作品集与职业发展

> 目标：学会构建有竞争力的设计作品集，规划 UI/UX 设计师的职业成长路径。

---

## 14.1 设计作品集的本质

**作品集不是作品展示，而是思维过程的展示。**

招聘方真正想看的是：
- 你如何发现问题？
- 你如何思考解决方案？
- 你如何验证设计决策？
- 你如何与团队协作？

> "任何人都可以学会使用 Figma，但能系统解决问题的设计师很少。"

---

## 14.2 作品集的基本要求

### 数量与质量

```
❌ 数量 vs ✅ 质量

  20 个简单界面练习  <  3 个深度案例

招聘方平均花 3-5 分钟看一份作品集，
深度比数量更重要

推荐：3-5 个精选案例，每个案例完整展示设计过程
```

### 案例类型搭配

```
理想的作品集组合：

1. 完整项目案例（最重要）：
   真实项目或自选题，从研究到交付的完整过程

2. 专项能力展示：
   • 视觉设计（纯 UI 美化）
   • 用户研究（访谈、分析报告）
   • 交互原型（复杂交互演示）

3. 竞品重设计（Side Project）：
   选一个现有产品的痛点，提出并设计改进方案
```

---

## 14.3 一个完整案例的结构

### 案例撰写框架（STAR-D）

```
S - Situation（背景）：项目背景，你的角色，面临的挑战
T - Task（任务）：你需要解决的问题是什么
A - Action（行动）：你做了什么（研究、方案、原型、测试）
R - Result（结果）：设计带来了什么改变（数据更好）
D - Design（设计展示）：最终的视觉成果
```

### 案例页面结构

```
封面
  ├── 项目名称
  ├── 一句话概括（解决了什么问题）
  └── 角色 / 时间 / 工具

1. 项目背景（1-2段）
   为什么这个项目存在？目标用户是谁？

2. 我的角色（说清楚你贡献了什么）
   "我负责用户研究和iOS端设计，参与了Android设计"

3. 发现问题（研究阶段）
   • 用户访谈关键发现
   • 竞品分析
   • 数据洞察
   → 最终定义的核心问题是什么？

4. 设计过程
   • 早期草图/概念探索
   • 关键设计决策和依据
   • 迭代对比（改之前 vs 改之后，说明原因）

5. 最终方案
   • 高保真界面展示
   • 核心功能演示（GIF或可交互原型）
   • 设计系统/组件（如有）

6. 结果与影响
   • 可用性测试结果
   • 上线后数据（如有：转化率+15%、NPS+20等）
   • 反思：如果重来会怎么做？

7. 学到了什么（可选，但加分）
   "这个项目让我学到了……"
```

## 14.4 如何自选案例题目

没有真实工作经验时，选择自选题/竞品重设计：

### 好的自选题方向

**竞品重设计（Redesign）**：
```
选择一个你认为有明显用户体验问题的 App
做用户研究（真实访谈）
提出并设计改进方案

注意：要基于研究，不要只是换颜色！
"我觉得美观"不是改的理由
```

**垂直场景解决方案**：
```
找一个特定人群的真实痛点：
• 独居老人如何使用社区服务
• 大学生如何管理兼职收入
• 健身新手如何坚持运动计划

从痛点出发，设计一个 App 解决方案
（不需要非常复杂，3-5个核心页面即可）
```

**选题评估标准**：
```
✓ 有真实的用户痛点（可以采访到真实用户）
✓ 问题定义清晰（不是"做一个更好的 X"）
✓ 解决方案可以用设计体现
✓ 你有能力进行深度研究
```

## 14.5 作品集展示形式

### 平台选择

| 平台 | 特点 | 适合 |
|------|------|------|
| **Behance** | 社区最大，浏览量高 | 视觉设计展示 |
| **Notion** | 排版自由，链接分享方便 | 深度案例文档 |
| **个人网站** | 完全自定义，最专业 | 有一定经验后 |
| **PDF 文件** | 方便投递简历附件 | 初入行时 |
| **Figma 链接** | 原型交互最真实 | 展示交互细节 |

**推荐组合**：Notion 深度案例 + Figma 原型 + PDF 版本备用

### 封面设计原则

```
好的作品集封面：
  ✓ 3秒内传达核心信息
  ✓ 高质量的界面截图或 Mockup
  ✓ 案例名称和一句话描述
  ✓ 视觉上有层次感

常见问题：
  ✗ 文字太多，不知道重点在哪
  ✗ 截图太小，看不清楚
  ✗ 纯文字，没有视觉展示
```

### 设备 Mockup

将界面放入设备框（iPhone/MacBook 等），增加真实感：

```
免费 Mockup 资源：
  • Mockuphone.com
  • DeviceMockups.com
  • Shots.so
  • Figma 社区搜索"device mockup"
```

## 14.6 简历与求职

### 设计师简历规范

**页面结构**：
```
[姓名]               [联系方式：邮件/手机]
[一句话定位：2年UI/UX设计经验，专注移动端B2C产品]

作品集链接：[URL]

工作经验
─────────────────────────────────────────
[公司名]  [职位]  [时间]
• 负责[产品名]的iOS端UI/UX设计，覆盖[用户量]
• 设计[核心功能]，用户留存率提升[X]%
• 独立完成从用户研究到设计交付的全流程

教育背景  技能清单  （放最后）
```

**简历注意事项**：
```
• 每段经历用数据量化（"重新设计了登录流程，弃单率降低30%"）
• 技能项要诚实（不要写"精通 PS/AI"，如果你只是会用）
• 一页纸（应届或经验 < 3年）
• PDF 格式，文件名：姓名_岗位_日期.pdf
• 作品集链接放最醒目的位置
```

### 求职平台

```
国内：
  • BOSS直聘（推荐）
  • 拉勾网（互联网职位多）
  • 猎聘（中高端）
  • 领英（LinkedIn，外企）

国外（远程/海外）：
  • LinkedIn
  • Glassdoor
  • AngelList（初创公司）
  • Dribbble Jobs
  • Behance Jobs
```

## 14.7 面试准备

### 作品集讲解（Portfolio Walkthrough）

面试中讲解作品集是关键环节，练习以下结构：

```
每个案例讲解控制在 5-8 分钟：

1. 项目背景（30秒）
   "这是一个解决[问题]的项目，用户是[谁]"

2. 我的贡献（30秒）
   "我在这个项目中负责[X Y Z]"

3. 设计过程（3分钟）
   "我们先做了用户研究，发现[关键洞察]，
    基于这个我们提出了[方案]，
    中间做了[测试]发现[问题]，最后调整为[方案2]"

4. 最终设计（2分钟）
   演示核心功能，讲解关键设计决策

5. 结果（30秒）
   "上线后[数据变化]，或测试显示[结果]"
```

### 常见面试问题准备

| 问题 | 回答思路 |
|------|---------|
| "介绍你最满意的一个设计案例" | 用 STAR 框架，强调过程和决策 |
| "你遇到过最难的设计挑战是什么？" | 诚实描述困难，重点说你如何解决 |
| "如果开发说你的设计实现不了，怎么办？" | 理解技术限制，寻找等效替代方案 |
| "如果产品/老板不采纳你的设计，怎么办？" | 用数据和用户研究支撑，求同存异 |
| "你如何评估设计的好坏？" | 用户目标达成率、可用性测试、数据指标 |
| "你怎么跟上设计趋势？" | 关注哪些博客/社区，有何独立判断 |

### 设计测试（Design Challenge）

部分公司会在面试中给出设计题：

```
常见形式：
  • 48小时作业：回家完成，提交 Figma 文件
  • 现场白板设计：30-60分钟完成一个设计

白板设计技巧：
  1. 先澄清需求（用户是谁？成功标准是什么？）
  2. 出声思考，让面试官知道你的思路
  3. 先画流程/架构，再画界面
  4. 主动讲解设计决策的原因
  5. 结尾说"如果有更多时间，我还会……"
```

## 14.8 职业成长路径

### 设计师级别

```
初级设计师（0-2年）
  • 掌握工具，执行明确的设计任务
  • 理解设计基础原则
  • 独立完成单个页面/功能设计

中级设计师（2-5年）
  • 独立负责完整功能模块
  • 有用户研究和数据分析能力
  • 能推动设计系统建设

高级设计师（5年+）
  • 负责产品级别的设计策略
  • 影响产品方向
  • 带领和指导初级设计师

设计主管/总监
  • 团队管理，设计文化建设
  • 与业务深度结合
  • 设计系统和设计语言决策
```

## 14.9 新手第一年的建议

### 90 天快速成长计划

```
第 1 个月：基础夯实
  • 完成本教材前 8 章，做完所有作业
  • 每天临摹 1 个优秀界面（理解，不只是模仿）
  • 注册 Figma，熟悉基础操作

第 2 个月：实战项目
  • 选定 1 个自选题，进行完整设计过程
  • 完成本教材后 6 章
  • 开始建立第一个作品集案例

第 3 个月：求职准备
  • 完善 2-3 个案例作品集
  • 修改简历，准备面试
  • 开始投递实习/初级岗位

在校学生：争取设计相关实习（哪怕是小公司）
应届毕业生：明确是走 UI 设计还是 UX 研究方向
转行者：保留现有领域知识，结合设计（如医疗UX、金融UX）
```

## 本章实践作业（最终项目）

**最终作业：完整的自选题设计项目**

要求：
1. 选定一个有真实用户痛点的领域（不能只是换颜色的竞品改版）
2. 采访至少 3 位目标用户，整理研究发现
3. 创建用户画像和用户旅程地图
4. 绘制信息架构和用户流程
5. 完成低保真线框图并进行一轮测试
6. 基于测试反馈优化，制作高保真界面（至少 8 个关键页面）
7. 设计组件库（核心组件）
8. 制作可交互原型
9. 整理为作品集案例（Notion 或 PDF）

**评估标准**：
- 研究是否扎实（有真实访谈？）
- 设计决策是否有依据？
- 视觉是否达到行业水平？
- 案例故事是否讲述清楚？

## 本章小结

| 概念 | 要点 |
|------|------|
| 作品集本质 | 展示思维过程，不只是成品界面 |
| 案例结构 | 背景→问题→研究→过程→方案→结果 |
| 数量质量 | 3-5个精深案例 > 20个表面练习 |
| 简历要点 | 用数据量化成果，链接放醒目位置 |
| 面试技巧 | 讲清楚"为什么这样设计" |
| 成长路径 | 初级执行→中级独立→高级策略 |

## 教材结语

恭喜你完成了这套 UI/UX 设计教材的学习！

设计是一门实践的学科，理论学完之后，**真正的成长来自于不断地做、测试、反思和迭代**。

记住：

> 每一个优秀的设计师，都曾有过糟糕的第一稿。
> 区别只在于，他们持续做了下去。

**你的下一步**：
1. 完成所有章节的实践作业
2. 完成最终的完整项目
3. 把作品集展示给真实用户和潜在雇主
4. 获取反馈，持续改进

祝你在 UI/UX 设计领域取得成功！
