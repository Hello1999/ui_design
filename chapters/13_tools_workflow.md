# 第 13 章：工具与工作流

> 目标：掌握专业的设计工作流，了解主流设计工具，学会规范地向开发团队交付设计稿。

---

## 13.1 设计工具全景

### 主流设计工具对比

| 工具 | 平台 | 特点 | 适用场景 |
|------|------|------|---------|
| **Figma** | Web/macOS/Win | 协作、组件、原型一体化 | 首选推荐，行业标准 |
| **Sketch** | macOS | 稳定、插件丰富 | 苹果生态设计师 |
| **Adobe XD** | Win/macOS | 与 Adobe 套件集成 | 已停止更新（不推荐新学）|
| **Axure** | Win/macOS | 复杂条件逻辑交互 | B端复杂原型 |
| **ProtoPie** | Win/macOS | 高级动效和传感器 | 动效演示 |
| **Framer** | Web | 代码级交互 | 开发者友好 |

**结论**：优先学 Figma，它是行业事实标准。

### 辅助工具

| 类别 | 工具 | 用途 |
|------|------|------|
| 用户研究 | FigJam、Miro | 亲和图、用户旅程、头脑风暴 |
| 用户测试 | Maze、Useberry | 远程可用性测试 |
| 素材资源 | Unsplash、Pexels | 免费高质量图片 |
| 图标库 | Feather、Iconoir、SF Symbols | 免费开源图标 |
| 插画库 | Storyset、unDraw、Blush | 免费可定制插画 |
| 字体 | Google Fonts | 免费 Web 字体 |
| 色彩 | Coolors、Huemint | 配色生成 |
| 原型分享 | Figma Link、Zeplin | 与开发/客户分享 |

## 13.2 Figma 高效工作流

### 文件组织规范

```
项目文件结构建议：

📁 [项目名]
├── 🎨 00_Design System（组件库和 Token）
├── 📋 01_Research（用户研究产出）
│   ├── Persona
│   ├── Journey Map
│   └── Competitive Analysis
├── 📐 02_Wireframes（线框图）
├── 🖌️ 03_UI Design（高保真界面）
│   ├── iOS 版
│   └── Android 版（如有）
├── 🔗 04_Prototype（可交互原型链接）
└── 📦 05_Handoff（交付文件）
    ├── 设计规范文档
    └── 切图资产
```

### Pages 组织

在 Figma 的 Pages 中合理组织：

```
Page 1: Cover（封面，项目名称、版本、更新日期）
Page 2: Design System（组件库）
Page 3: User Flow（用户流程图）
Page 4: Wireframes（线框图）
Page 5: Final Design - iOS（最终设计稿）
Page 6: Final Design - Web（如有）
Page 7: Handoff Notes（交付说明）
```

### 图层命名规范

```
好的命名：清晰、语义化、用 / 分层
  button/primary/default
  icon/home/24px
  text/body/regular

坏的命名：
  Rectangle 1
  Group 3
  Copy of button
```

**批量重命名**：选中多个图层 → Cmd+R（Mac）→ 批量重命名

## 13.3 设计规范与交付

### 设计交付物（Deliverables）清单

**给产品经理/项目团队**：
- [ ] 用户流程图（User Flow）
- [ ] 高保真设计稿（含所有页面和状态）
- [ ] 可交互原型链接

**给开发工程师**：
- [ ] 设计规范文档（颜色、字体、间距）
- [ ] 标注稿（尺寸、间距、颜色值）
- [ ] 切图资产（图标、插图）
- [ ] 组件说明（交互行为描述）

### Figma 开发模式（Dev Mode）

Figma 的 Dev Mode 专门为开发者设计：

```
切换到 Dev Mode：右上角点击 </> 图标

开发者可以看到：
  • 自动生成的 CSS/iOS/Android 代码
  • 精确的尺寸和间距数值
  • 颜色的 HEX/RGB 值
  • 字体属性
  • 组件的变体信息
```

### 标注规范

标注需要提供给开发的信息：

```
尺寸标注：
  元素宽度：358px
  元素高度：48px
  圆角：8px

间距标注：
  元素间距：16px
  内边距：12px 16px（上下12，左右16）

颜色标注：
  背景色：#4299E1
  文字色：#FFFFFF
  边框色：#E8E8E8

字体标注：
  字体：SF Pro / PingFang SC
  字号：16px
  字重：Semibold（600）
  行高：24px
  字间距：0
```

## 13.4 切图规范

### 需要切图的元素

```
需要切图：
  ✓ 自定义图标（SVG 格式）
  ✓ 品牌 Logo
  ✓ 插画/装饰图
  ✓ 背景纹理

不需要切图（代码实现）：
  ✗ 纯色矩形（用 backgroundColor）
  ✗ 圆角（用 borderRadius）
  ✗ 文字（用代码渲染）
  ✗ 阴影（用 boxShadow）
```

### 图标切图规范

```
iOS：
  @1x：24×24px（普通屏）
  @2x：48×48px（视网膜屏，大多数 iPhone）
  @3x：72×72px（高分辨率屏）

Android：
  mdpi：24×24dp（基准）
  hdpi：36×36dp
  xhdpi：48×48dp
  xxhdpi：72×72dp
  xxxhdpi：96×96dp

Web：
  SVG 格式（矢量，无需多倍图）
  PNG 备用（提供 1x 和 2x）
```

### Figma 导出设置

```
选中图层 → 右下角 Export 面板 → 添加导出配置

图标（SVG）：
  格式：SVG

图片（PNG）：
  格式：PNG
  Scale: 1x, 2x, 3x
  后缀: @1x, @2x, @3x

App Icon：
  按 iOS/Android 规范提供所有尺寸
```

## 13.5 版本管理

### Figma 版本历史

```
保存版本：Ctrl/Cmd + Alt + S → 添加版本说明
查看历史：File → Version History

版本命名规范：
  v1.0 - 初始设计稿（日期）
  v1.1 - 修改登录流程
  v2.0 - 大改版，新导航结构

关键节点必须命名版本！
避免："最终版"、"最终确认版"、"最终最终版"（血泪教训）
```

### 分支（Branch）协作

大型项目使用 Figma 分支功能：

```
主分支（Main）：稳定的已发布版本

分支 A：设计师 A 的改版探索
  ├── 修改导航结构
  └── 验证通过后合并回 Main

分支 B：设计师 B 的新功能设计
  ├── 评论页重设计
  └── 独立进行，不影响主分支
```

## 13.6 设计评审（Design Review）

### 自我评审清单（提交前检查）

**视觉层面**
- [ ] 所有文字对比度达标
- [ ] 间距使用 8pt 系统，无随意间距
- [ ] 所有元素对齐整齐
- [ ] 颜色使用统一的 Token/颜色库
- [ ] 图标风格一致

**内容层面**
- [ ] 所有占位符文字已替换为真实内容
- [ ] 文案无错别字，符合产品语气
- [ ] 所有状态（空/加载/错误）都有设计

**交互层面**
- [ ] 所有关键流程可以走通
- [ ] 手势和交互有文字说明
- [ ] 与开发确认过技术可行性

**规范层面**
- [ ] 使用了设计系统中的组件（非独立绘制）
- [ ] 图层命名规范
- [ ] 文件组织整洁

### 与团队协作的技巧

**清晰地描述设计决策**：
```
✗ 不好："我觉得这样比较好看"
✓ 好的："这个设计的依据是：
         1. 用户研究显示用户希望在列表页直接预览
         2. 竞品 A 也采用了类似的展示方式
         3. 可以减少用户进入详情页的点击次数"
```

**接收反馈的正确姿态**：
```
设计被质疑时：
  • 先确认对方的问题（"你的意思是……对吗？"）
  • 解释设计决策背后的逻辑
  • 如果对方有合理点，承认并讨论改进方向
  • 不要"因为我喜欢"来辩护设计决策
```

## 13.7 AI 工具辅助设计

### 2024-2026 AI 设计工具

| 工具 | 用途 |
|------|------|
| **Figma AI** | 自动布局、内容生成、图层整理 |
| **Midjourney / Stable Diffusion** | 生成概念图、情绪板 |
| **ChatGPT / Claude** | 文案撰写、用户研究分析 |
| **Galileo AI** | 从文字描述生成 UI |
| **UXpilot** | AI 生成用户故事和流程图 |

**AI 的正确使用方式**：
```
AI 擅长：
  • 快速生成初步方案（需人工筛选和优化）
  • 辅助撰写文案
  • 分析用户反馈
  • 生成概念图

AI 不能替代：
  • 用户研究和洞察理解
  • 设计决策和判断
  • 与团队的沟通协作
  • 最终的审美判断
```

## 13.8 设计师的效率技巧

### Figma 必备插件

| 插件 | 用途 |
|------|------|
| **Unsplash** | 快速插入免费图片 |
| **Iconify** | 30万+免费图标库 |
| **Content Reel** | 随机填充真实文案/头像 |
| **Contrast** | 检查颜色对比度 |
| **Stark** | 无障碍检查套件 |
| **Auto Flow** | 自动绘制流程图箭头 |
| **Figma to HTML** | 导出代码（参考用）|

### 提升工作效率的习惯

```
1. 先整体后局部：先确定信息架构，再做视觉细节
2. 组件优先：用到的元素先做成组件，避免后期不一致
3. 真实内容：从第一天就用真实文案，避免"Lorem ipsum"
4. 定期整理文件：保持图层整洁，便于协作和后期修改
5. 快捷键练习：每周学 3 个新快捷键，积累下来效率翻倍
6. 建立素材库：整理常用图标、颜色、组件，下次直接复用
```
