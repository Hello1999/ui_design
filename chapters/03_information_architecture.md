# 第 03 章：信息架构

> 目标：理解如何组织和结构化信息，让用户能轻松找到他们需要的内容。

---

## 3.1 什么是信息架构

**信息架构（Information Architecture，IA）** 是对信息内容的组织、标注、导航与搜索系统的设计，目的是帮助用户理解"我在哪里"、"我能去哪里"、"我找到了什么"。

### 信息架构的三个核心问题

```
用户进入一个页面时，必须能回答：

1. 我在哪里？         → 当前位置（面包屑、高亮导航）
2. 我能做什么？       → 可用操作（按钮、链接、功能）
3. 我下一步去哪里？   → 导航与流程引导
```

### 信息架构的四大系统

| 系统 | 作用 | 示例 |
|------|------|------|
| **组织系统** | 如何对内容分类 | 按主题/时间/地域分类 |
| **标签系统** | 如何命名内容 | 菜单名称、按钮文案 |
| **导航系统** | 如何在内容间移动 | 顶导航、侧边栏、标签栏 |
| **搜索系统** | 如何检索内容 | 搜索框、筛选器 |

---

## 3.2 组织内容的方式

### LATCH 模型（五种分类方式）

| 维度 | 英文 | 适用场景 | 示例 |
|------|------|---------|------|
| **位置** | Location | 地理相关内容 | 地图、同城服务 |
| **字母** | Alphabet | 词典、目录 | A-Z 联系人 |
| **时间** | Time | 历史、日程 | 动态 Feed、新闻 |
| **类别** | Category | 产品、服务 | 电商分类 |
| **层级** | Hierarchy | 复杂系统 | 文件管理器 |

### 内容分类原则

**相互独立，完全穷举（MECE 原则）**

```
✗ 错误的分类（重叠）：
  ├── 电子产品
  ├── 手机
  └── 数码相机

✓ 正确的分类（独立）：
  ├── 手机与平板
  ├── 电脑与外设
  ├── 摄影摄像
  └── 家用电器
```

## 3.3 导航模式

### 移动端导航模式

**1. 标签栏导航（Tab Bar）**
```
适用：3-5 个一级功能，用户频繁切换

┌─────────────────────────────┐
│         内容区域             │
│                             │
│                             │
├─────┬─────┬─────┬─────┬────┤
│ 首页 │ 探索 │ 发布 │ 消息 │ 我的│
└─────┴─────┴─────┴─────┴────┘

优点：直接可见，切换成本低
缺点：占用底部空间，最多 5 项
```

**2. 汉堡菜单（Hamburger Menu）**
```
适用：功能较多，非核心功能收纳

┌─────────────────────────────┐
│ ≡  [Logo]              🔍  │  ← 点击 ≡ 展开侧边菜单
├─────────────────────────────┤
│ 内容区域                     │
└─────────────────────────────┘

优点：节省空间，可放更多功能
缺点：隐藏导致发现率低，点击成本高
```

**3. 顶部标签导航**
```
适用：同级内容切换，如资讯分类

┌─────────────────────────────┐
│ 推荐 | 关注 | 视频 | 热榜     │  ← 可横向滑动
├─────────────────────────────┤
│ 内容 Feed                   │
└─────────────────────────────┘
```

**4. 卡片/宫格导航**
```
适用：工具类 App，功能入口并列

┌────────┬────────┬────────┐
│  扫一扫 │  付款  │  收款   │
├────────┼────────┼────────┤
│  转账  │  理财  │  卡券   │
└────────┴────────┴────────┘
```

### Web 端导航模式

**1. 顶部水平导航（最常见）**
```
[Logo]  产品  方案  案例  关于  [注册] [登录]
────────────────────────────────────────────
                  内容区域
```

**2. 侧边栏导航（适合复杂后台）**
```
┌──────────┬─────────────────────────┐
│          │   页面标题               │
│  导航     │                         │
│  ───     │   内容区域               │
│  首页    │                         │
│  数据    │                         │
│  设置    │                         │
└──────────┴─────────────────────────┘
```

## 3.4 心智模型

**心智模型** 是用户对系统工作方式的主观理解。设计应当匹配用户的已有心智模型。

### 示例：购物车的心智模型

用户期望购物车的行为：
- 东西放进去不会自动消失
- 可以修改数量
- 可以暂时保存（不急着结算）

**违背心智模型的设计**：
- 超过 24 小时自动清空购物车 → 用户沮丧
- 结算后无法查看历史订单 → 用户困惑

**匹配心智模型的设计**：
- 参考现实世界中"购物车/篮子"的行为
- 使用用户熟悉的隐喻（图标、文案、流程）

### 尼尔森十大可用性原则（与 IA 相关部分）

| 原则 | 说明 |
|------|------|
| 系统状态可见 | 用户知道系统在做什么（进度条、加载状态）|
| 匹配现实世界 | 使用用户熟悉的词汇和概念 |
| 用户控制自由 | 提供撤销、返回、取消 |
| 一致性与标准 | 相同操作始终有相同结果 |
| 认知而非记忆 | 减少用户需要记忆的信息量 |

## 3.5 卡片分类法（Card Sorting）

卡片分类是研究用户如何组织信息的方法，帮助你设计符合用户心智模型的导航结构。

### 操作步骤

**1. 开放式卡片分类**（探索用户分类逻辑）
```
准备：把所有内容/功能写在卡片上（20-40张）

让用户：
  1. 把卡片分成若干组
  2. 给每组起名字
  3. 解释分类逻辑

分析：找出用户分类中的共同模式
```

**2. 封闭式卡片分类**（验证已有分类方案）
```
准备：预先定义好分类（导航类目）

让用户：
  把每张内容卡片放入对应类目

分析：某内容被放错类目的频率 → 导航标签需要优化
```

**工具推荐**：
- Optimal Workshop（专业卡片分类工具）
- FigJam（虚拟便利贴，简易版）
- 真实便利贴（线下操作）

## 3.6 网站地图（Site Map）

网站地图是信息架构的可视化文档，展示产品的页面层级结构。

### 示例：电商 App 网站地图

```
电商 App
├── 首页
│   ├── 搜索
│   ├── Banner（活动专题）
│   └── 商品推荐列表
├── 分类
│   ├── 一级分类
│   └── 二级分类 → 商品列表
├── 购物车
│   └── 结算页
│       ├── 地址选择
│       ├── 支付方式
│       └── 订单确认
├── 我的订单
│   ├── 待付款
│   ├── 待发货
│   ├── 待收货
│   └── 已完成 → 评价页
└── 个人中心
    ├── 账户信息
    ├── 收货地址
    ├── 优惠券
    └── 设置
```

### 网站地图规范

- 使用方框表示页面，线段表示导航关系
- 区分不同类型节点：页面 / 弹窗 / 外链
- 标注用户权限（登录/未登录可见的页面）

## 3.7 任务流程图（Task Flow）

任务流程图描述用户完成特定任务的操作步骤，比网站地图更聚焦于行为流程。

### 符号规范

```
圆角矩形  = 页面/屏幕
菱形      = 判断/条件
矩形      = 操作/步骤
圆形      = 开始/结束
箭头      = 流程方向
```

### 示例：用户注册流程

```
  ●开始
   ↓
  [打开注册页]
   ↓
  [输入手机号]
   ↓
  ◇ 手机号已注册？
  ├─ 是 → [提示：该号码已注册，去登录] → [登录页]
  └─ 否 ↓
  [发送验证码]
   ↓
  [输入验证码]
   ↓
  ◇ 验证码正确？
  ├─ 否 → [提示错误，重新输入]（循环）
  └─ 是 ↓
  [设置密码]
   ↓
  [注册成功 → 进入首页]
   ↓
  ●结束
```

## 3.8 IA 设计的常见错误

| 错误 | 问题 | 解决方案 |
|------|------|---------|
| 导航层级过深 | 用户需要点击 5 次才能到达目标 | 控制在 3 层以内 |
| 功能入口过多 | 首页有 20 个功能入口 | 优先展示核心功能（最多 8 个） |
| 标签名称模糊 | 菜单叫"工具箱"，用户不知道里面有什么 | 使用动词+名词的具体描述 |
| 导航分类不一致 | 同类内容散布在多个分类下 | 严格执行 MECE 分类原则 |
| 搜索与导航不互补 | 有搜索但无法按类目浏览 | 搜索与导航各司其职 |

## 本章小结

|------|------|
| 信息架构 | 组织、标签、导航、搜索四大系统 |
