## 常见错误

### 视觉设计错误

#### 1. 对比度不足

```
❌ 错误：
background: #FFFFFF;
color: #CCCCCC; /* 对比度 1.6:1 */

✅ 正确：
background: #FFFFFF;
color: #6B7280; /* 对比度 4.6:1 */

检查工具：
- Figma插件：Contrast
- 网站：WebAIM Contrast Checker
```

#### 2. 间距不一致

```
❌ 错误：
margin: 15px;
padding: 18px;
gap: 13px;

✅ 正确（8点系统）：
margin: 16px; /* 8 × 2 */
padding: 24px; /* 8 × 3 */
gap: 12px; /* 8 × 1.5 */

使用：
- 设计令牌
- Auto Layout
- 网格系统
```

#### 3. 字体混乱

```
❌ 错误：
- 使用3种以上字体
- 字号随意（17px, 19px等）
- 行高不规则

✅ 正确：
- 最多2-3种字体
- 使用模块化比例
- 一致的行高系统
```

#### 4. 颜色过多

```
❌ 错误：
使用10+种颜色

✅ 正确：
- 主色：1-2种
- 功能色：4种（成功、警告、错误、信息）
- 中性色：灰度系统
- 强调色：1种

应用60-30-10规则
```

---

### 交互设计错误

#### 1. 缺少状态

```
❌ 错误：
只设计默认状态

✅ 正确：
设计所有状态：
- Default
- Hover
- Active/Pressed
- Focus
- Disabled
- Loading
- Error
- Success
```

#### 2. 触摸目标过小

```
❌ 错误：
移动端按钮 32x32px

✅ 正确：
最小 48x48px（iOS/Material Design）

检查：
- 所有可点击元素
- 按钮间距
- 链接大小
```

#### 3. 反馈不清晰

```
❌ 错误：
点击按钮无反馈

✅ 正确：
- 视觉变化（颜色、阴影）
- 加载指示
- 成功/失败提示
- 过渡动画
```

---

### 响应式错误

#### 1. 固定宽度

```
❌ 错误：
.container {
  width: 1200px;
}

✅ 正确：
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 16px;
}
```

#### 2. 不考虑内容伸缩

```
❌ 错误：
固定高度，内容溢出

✅ 正确：
- 使用 min-height
- 允许内容扩展
- 测试长文本
```

#### 3. 断点设置不当

```
❌ 错误：
只考虑iPhone尺寸

✅ 正确：
- 测试多种设备
- 基于内容设置断点
- 移动优先方法
```

## 进阶技巧

### 设计系统搭建

#### 1. 设计令牌架构

```css
/* 原始令牌（Primitive）*/
:root {
  --color-blue-500: #3B82F6;
  --color-gray-900: #111827;
  --spacing-4: 16px;
}

/* 语义令牌（Semantic）*/
:root {
  --color-primary: var(--color-blue-500);
  --color-text: var(--color-gray-900);
  --space-default: var(--spacing-4);
}

/* 组件令牌（Component）*/
.button {
  background: var(--color-primary);
  padding: var(--space-default);
}
```

#### 2. 组件库结构

```
原子设计（Atomic Design）：

Atoms（原子）：
- Button
- Input
- Icon
- Label

Molecules（分子）：
- Form Field (Label + Input)
- Search Bar (Input + Icon + Button)
- Card Header (Avatar + Text)

Organisms（有机体）：
- Navigation
- Form
- Card
- Modal

Templates（模板）：
- Page layouts
- Section structures

Pages（页面）：
- Specific instances
```

### CSS实现技巧

#### 1. 使用CSS变量

```css
/* 定义变量 */
:root {
  --primary: #3B82F6;
  --radius: 8px;
  --shadow: 0 2px 8px rgba(0,0,0,0.1);
}

/* 使用 */
.button {
  background: var(--primary);
  border-radius: var(--radius);
  box-shadow: var(--shadow);
}

/* 深色模式切换 */
@media (prefers-color-scheme: dark) {
  :root {
    --primary: #60A5FA;
  }
}
```

#### 2. 实用的CSS片段

```css
/* 平滑滚动 */
html {
  scroll-behavior: smooth;
}

/* 选中文本颜色 */
::selection {
  background: #3B82F6;
  color: white;
}

/* 图片优化 */
img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* 隐藏滚动条但保持功能 */
.scroll-container {
  scrollbar-width: none;
  -ms-overflow-style: none;
}
.scroll-container::-webkit-scrollbar {
  display: none;
}

/* 文本溢出省略 */
.truncate {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}

/* 多行省略 */
.line-clamp-2 {
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

## 设计思维

### 用户至上

```
设计时始终问自己：

✓ 这对用户有什么好处？
✓ 用户能理解吗？
✓ 操作是否简单直观？
✓ 有更好的方式吗？

避免：
❌ 为了设计而设计
❌ 炫技
❌ 忽视用户需求
❌ 过度复杂
```

### 从问题出发

```
设计流程：

1. 定义问题
   - 真正的问题是什么？
   - 为谁解决？
   - 为什么重要？

2. 探索方案
   - 头脑风暴
   - 参考案例
   - 快速草图

3. 验证想法
   - 制作原型
   - 用户测试
   - 收集反馈

4. 迭代优化
   - 分析数据
   - 改进方案
   - 再次测试
```

### 保持简洁

```
简化原则：

✓ 能删就删
✓ 能合并就合并
✓ 每个元素都有目的
✓ 提供默认值
✓ 渐进披露

示例：
❌ 复杂：10个选项，5个步骤
✅ 简洁：智能默认，1个步骤
```

```

1. 2-3个完整项目
   - 过程展示
   - 设计决策
   - 成果展示

2. 多样性

3. 真实案例优先
```
