# 第四章：排版设计

排版设计（Typography）是UI设计中最重要的技能之一。优秀的排版不仅能提升视觉美感，更能显著改善用户的阅读体验和信息获取效率。本章将深入探讨排版设计的原理、技巧和最佳实践。

### 4.1.2 字体分类

理解不同字体类型及其应用场景：

```css

    /* 特点：有装饰性笔画，传统、正式、易读 */
```

## 4.2 字体大小与比例

### 4.2.1 模块化比例系统

使用数学比例创建和谐的字体大小系统：

```css
/* 基于完美四度（4:3 = 1.333）的比例系统 */
:root {
    --ratio: 1.333;
    --base-size: 16px;

    /* 使用 calc() 计算各级字体大小 */
    --text-xs: calc(var(--base-size) / var(--ratio) / var(--ratio));    /* ~9px */
    --text-sm: calc(var(--base-size) / var(--ratio));                   /* ~12px */
    --text-base: var(--base-size);                                       /* 16px */
    --text-lg: calc(var(--base-size) * var(--ratio));                   /* ~21px */
    --text-xl: calc(var(--base-size) * var(--ratio) * var(--ratio));   /* ~28px */
    --text-2xl: calc(var(--base-size) * var(--ratio) * var(--ratio) * var(--ratio)); /* ~37px */
    --text-3xl: calc(var(--base-size) * var(--ratio) * var(--ratio) * var(--ratio) * var(--ratio)); /* ~49px */
}

/* 黄金比例系统（1.618） */
.golden-ratio-scale {
    --ratio: 1.618;
    --base: 16px;

    --h6: calc(var(--base) * 1);                    /* 16px */
    --h5: calc(var(--base) * var(--ratio));         /* ~26px */
    --h4: calc(var(--base) * var(--ratio) * var(--ratio));      /* ~42px */
    --h3: calc(var(--base) * var(--ratio) * var(--ratio) * var(--ratio)); /* ~68px */
}

/* 常见比例选择 */
.type-scales {
    /* 小三度 (1.2) - 适合内容密集的界面 */
    --minor-third: 1.2;

    /* 大三度 (1.25) - 平衡的选择 */
    --major-third: 1.25;

    /* 完全四度 (1.333) - 推荐用于 Web */
    --perfect-fourth: 1.333;

    /* 增强四度 (1.414) - 视觉冲击力强 */
    --augmented-fourth: 1.414;

    /* 完全五度 (1.5) - 戏剧性对比 */
    --perfect-fifth: 1.5;

    /* 黄金比例 (1.618) - 自然和谐 */
    --golden-ratio: 1.618;
}
```

## 4.3 行高与间距

### 4.3.1 行高（Line Height）原理

```css
/* 行高基础理论 */
.line-height-basics {
    /* 绝对值 - 不推荐，不会随字体缩放 */
    line-height: 24px;

    /* 相对值 - 推荐，会随字体大小缩放 */
    line-height: 1.5; /* 无单位，最灵活 */
    line-height: 150%; /* 百分比 */
    line-height: 1.5em; /* em 单位 */
}

/* 不同内容类型的行高建议 */
:root {
    /* 标题 - 较紧凑 */
    --line-height-tight: 1.1;
    --line-height-snug: 1.25;

    /* 正文 - 舒适阅读 */
    --line-height-normal: 1.5;
    --line-height-relaxed: 1.625;

    /* 长文本 - 更宽松 */
    --line-height-loose: 1.75;
    --line-height-extra-loose: 2;
}

/* 应用规则 */
h1, h2, h3 {
    line-height: var(--line-height-tight);
}

h4, h5, h6 {
    line-height: var(--line-height-snug);
}

p, li {
    line-height: var(--line-height-normal);
}

.article-content {
    line-height: var(--line-height-relaxed);
}

blockquote {
    line-height: var(--line-height-loose);
}

/* 根据字体大小动态调整行高 */
.dynamic-line-height {
    font-size: clamp(1rem, 0.9rem + 0.5vw, 1.125rem);
    /* 字体越大，行高比例可以越小 */
    line-height: calc(1.8 - 0.3 * (1vw / 10));
}
```

## 4.8 常见错误与解决方案

### 错误1：字体大小过小

```css
/* ❌ 错误：正文字体太小 */
.bad-example {
    font-size: 12px; /* 难以阅读 */
}

/* ✅ 正确：使用合适的字体大小 */
.good-example {
    font-size: 16px; /* 或 1rem */
    /* 移动端可以稍小 */
    @media (max-width: 768px) {
        font-size: 14px;
    }
}
```

### 错误2：行高不当

```css
/* ❌ 错误：行高太紧 */
.bad-line-height {
    line-height: 1; /* 行与行之间太挤 */
}

/* ❌ 错误：行高太松 */
.bad-line-height-loose {
    line-height: 2.5; /* 阅读时容易跳行 */
}

/* ✅ 正确：根据内容类型选择合适行高 */
.good-line-height-heading {
    line-height: 1.2; /* 标题 */
}

.good-line-height-body {
    line-height: 1.6; /* 正文 */
}
```

### 错误3：行长度过长

```css
/* ❌ 错误：行太长 */
.bad-line-length {
    max-width: 100%; /* 在大屏幕上难以阅读 */
}

/* ✅ 正确：限制行长度 */
.good-line-length {
    max-width: 70ch; /* 约70个字符宽度 */
    margin: 0 auto;
}
```

### 错误4：对比度不足

```css
/* ❌ 错误：对比度太低 */
.bad-contrast {
    color: #aaa;
    background: #fff;
    /* 对比度只有 2.3:1，不符合 WCAG 标准 */
}

/* ✅ 正确：确保足够对比度 */
.good-contrast {
    color: #333;
    background: #fff;
    /* 对比度 12.6:1，远超 WCAG AA 标准 */
}
```

### 错误5：忽略字体堆栈

```css
/* ❌ 错误：只指定一种字体 */
.bad-font-stack {
    font-family: 'Custom Font';
    /* 如果字体加载失败，会回退到默认字体 */
}

/* ✅ 正确：提供完整的字体堆栈 */
.good-font-stack {
    font-family: 'Custom Font', -apple-system, BlinkMacSystemFont,
                 'Segoe UI', sans-serif;
}
```

## 4.9 练习题

### 练习1：创建排版系统

创建一个完整的排版系统，包括：
- 字体大小变量（至少5个级别）
- 行高变量（3个级别）
- 字重变量（3-4个级别）
- 应用到标题和正文

### 练习2：优化文章页面

给定一个排版混乱的文章页面，进行以下优化：
- 改善字体大小和行高
- 限制行长度
- 增加段落间距
- 优化标题层级

### 练习3：创建卡片组件

设计一个产品卡片，要求：
- 标题最多显示2行，超出显示省略号
- 描述最多显示3行
- 价格使用等宽数字
- 按钮文字使用合适的字间距

### 练习4：响应式排版

创建一个响应式排版系统，要求：
- 使用 `clamp()` 实现流畅缩放
- 在不同屏幕尺寸下保持良好可读性
- 行长度自适应

### 练习5：可访问性改进

检查并改进以下方面：
- 文字与背景的对比度（至少4.5:1）
- 链接的可见性和可点击性
- 字体大小是否足够大
- 是否支持用户自定义字体大小

## 4.10 总结

排版设计是UI设计的基础，掌握以下核心原则：

1. **选择合适的字体**：根据内容类型和品牌调性选择字体
2. **建立比例系统**：使用数学比例创建和谐的字体大小
3. **优化行高**：根据字体大小和行长度调整行高
4. **控制行长度**：保持在45-75个字符（英文）或35-45个字符（中文）
5. **确保对比度**：文字与背景对比度至少4.5:1
6. **响应式设计**：在不同设备上保持良好可读性
7. **注重细节**：字间距、词间距、对齐方式等细节都很重要
8. **考虑可访问性**：确保所有用户都能舒适阅读

通过持续练习和实践，你将能够创造出既美观又实用的排版设计，为用户提供卓越的阅读体验。
