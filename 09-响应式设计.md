## 响应式设计原则

### 移动优先 (Mobile First)

**原理：** 从最小屏幕开始设计，逐步增强到大屏幕

```css
/* ✅ 移动优先方法 */
.container {
  /* 移动端样式（基础）*/
  padding: 16px;
  font-size: 16px;
}

@media (min-width: 768px) {
  /* 平板样式（增强）*/
  .container {
    padding: 32px;
    font-size: 18px;
  }
}

@media (min-width: 1024px) {
  /* 桌面样式（再增强）*/
  .container {
    padding: 48px;
    font-size: 20px;
  }
}
```

**优势：**
```
✓ 强制优先考虑核心内容
✓ 更好的性能（移动端加载更少代码）
✓ 渐进增强而非优雅降级
✓ 符合移动流量占主导的趋势
```

### 流式布局 (Fluid Layout)

```css
/* 使用百分比而非固定像素 */
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 5%; /* 百分比内边距 */
}

.column {
  width: 100%;
}

@media (min-width: 768px) {
  .column {
    width: 48%; /* 百分比宽度 */
  }
}
```

### 弹性媒体 (Flexible Media)

```css
/* 图片自适应 */
img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* 视频自适应 */
video {
  max-width: 100%;
  height: auto;
}

/* 响应式 iframe */
.video-container {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 比例 */
  height: 0;
  overflow: hidden;
}

.video-container iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
```

## 断点设计

### 常用断点

```css
:root {
  /* 断点定义 */
  --breakpoint-sm: 640px;   /* 手机横屏 */
  --breakpoint-md: 768px;   /* 平板竖屏 */
  --breakpoint-lg: 1024px;  /* 笔记本 */
  --breakpoint-xl: 1280px;  /* 桌面 */
  --breakpoint-2xl: 1536px; /* 大屏 */
}

/* 移动端（默认）*/
.container {
  padding: 16px;
}

/* 小手机横屏 */
@media (min-width: 640px) {
  .container {
    padding: 20px;
  }
}

/* 平板 */
@media (min-width: 768px) {
  .container {
    padding: 32px;
  }
}

/* 笔记本 */
@media (min-width: 1024px) {
  .container {
    padding: 40px;
  }
}

/* 桌面 */
@media (min-width: 1280px) {
  .container {
    padding: 48px;
    max-width: 1200px;
    margin: 0 auto;
  }
}

/* 大屏 */
@media (min-width: 1536px) {
  .container {
    max-width: 1400px;
  }
}
```

### 基于内容的断点

```
不要只依赖设备尺寸：
✓ 观察内容何时需要调整
✓ 在布局变得拥挤时添加断点
✓ 测试实际设备

示例：
- 导航菜单何时需要折叠？
- 多列布局何时应该变成单列？
- 字体大小何时需要调整？
```

### 懒加载

```html
<!-- 原生懒加载 -->
<img src="image.jpg" loading="lazy" alt="描述" />

<!-- 占位符 -->
<img
  src="placeholder-low.jpg"
  data-src="image-full.jpg"
  class="lazy"
  alt="描述"
/>

<script>
// Intersection Observer 懒加载
const images = document.querySelectorAll('.lazy');

const imageObserver = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.remove('lazy');
      imageObserver.unobserve(img);
    }
  });
});

images.forEach(img => imageObserver.observe(img));
</script>
```

## 常见问题与解决

### 1. iOS 输入框缩放

```html
<!-- 问题：输入框聚焦时页面缩放 -->
<!-- 解决：设置最小字号16px -->
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">

<style>
input, textarea {
  font-size: 16px; /* 至少16px */
}
</style>
```

### 2. 移动端 100vh 问题

```css
/* 问题：100vh包含地址栏，导致滚动 */

/* 解决1：使用 dvh（动态视口高度）*/
.full-height {
  height: 100dvh;
}

/* 解决2：JavaScript 计算 */
.full-height {
  height: 100vh;
  height: calc(var(--vh, 1vh) * 100);
}
```

```javascript
// 设置真实视口高度
function setVH() {
  const vh = window.innerHeight * 0.01;
  document.documentElement.style.setProperty('--vh', `${vh}px`);
}

setVH();
window.addEventListener('resize', setVH);
```

### 3. 水平滚动条

```css
/* 防止意外的水平滚动 */
html, body {
  overflow-x: hidden;
}

* {
  /* 确保元素不超出容器 */
  max-width: 100%;
}
```

### 4. 图片比例保持

```css
/* 保持图片比例 */
.image-container {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 */
  height: 0;
  overflow: hidden;
}

.image-container img {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```
