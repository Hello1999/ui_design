# 第六章：视觉层级

视觉层级（Visual Hierarchy）是UI设计中最关键的概念之一。它决定了用户如何浏览页面、理解信息的重要性顺序，以及完成任务的效率。优秀的视觉层级设计能够引导用户的注意力，创造清晰的信息架构，并提升整体用户体验。

## 6.1 视觉层级基础理论

### 6.1.1 什么是视觉层级

视觉层级是通过视觉元素（大小、颜色、对比度、位置、形状等）来组织和呈现信息的方式，使得某些元素比其他元素更突出或更重要。

```html
</body>
</html>
```

## 6.6 常见错误与最佳实践

### 错误1：缺乏对比

```css
/* ❌ 错误：所有元素大小相似 */
.bad-contrast {
    /* 标题 */
    font-size: 1.125rem;
    font-weight: 500;
}

.bad-contrast p {
    /* 正文 */
    font-size: 1rem;
    font-weight: 400;
}
/* 对比度不足，难以区分层级 */

/* ✅ 正确：明显的大小对比 */
.good-contrast h1 {
    font-size: 3rem;
    font-weight: 700;
}

.good-contrast p {
    font-size: 1rem;
    font-weight: 400;
}
/* 清晰的视觉层级 */
```

### 错误2：过度设计

```css
/* ❌ 错误：太多视觉层级 */
.bad-hierarchy {
    /* 试图让所有内容都突出 */
}

.bad-hierarchy h1 {
    font-size: 4rem;
    font-weight: 900;
    color: #ff0000;
    text-shadow: 2px 2px 4px rgba(0,0,0,0.5);
    animation: pulse 1s infinite;
}

.bad-hierarchy h2 {
    font-size: 3rem;
    font-weight: 800;
    color: #0000ff;
    text-decoration: underline;
}
/* 结果：没有重点，视觉混乱 */

/* ✅ 正确：适度的层级 */
.good-hierarchy h1 {
    font-size: 2.5rem;
    font-weight: 700;
    color: #1a1a1a;
    margin-bottom: 1rem;
}

.good-hierarchy h2 {
    font-size: 1.75rem;
    font-weight: 600;
    color: #333;
    margin-bottom: 0.75rem;
}
/* 清晰简洁，重点突出 */
```

### 错误3：忽视空白

```css
/* ❌ 错误：缺少空白空间 */
.bad-spacing {
    margin: 0;
    padding: 0;
}

.bad-spacing * {
    margin: 0.25rem;
    padding: 0.25rem;
}
/* 拥挤，难以阅读 */

/* ✅ 正确：适当的空白 */
.good-spacing {
    max-width: 800px;
    margin: 0 auto;
    padding: 2rem;
}

.good-spacing h2 {
    margin-top: 3rem;
    margin-bottom: 1.5rem;
}

.good-spacing p {
    margin-bottom: 1.5rem;
    line-height: 1.6;
}
/* 呼吸感，易于阅读 */
```

## 6.7 练习题

### 练习1：创建信息卡片层级
设计一个信息卡片，包含：
- 图标（最高层级）
- 标题（次高层级）
- 描述（中等层级）
- 元数据（最低层级）

要求使用大小、颜色、间距创建清晰层级。

### 练习2：重设计低层级页面
给定一个视觉层级不清晰的页面，进行重设计：
- 识别最重要的3个元素
- 使用视觉层级原则突出它们
- 减少次要元素的视觉重量

### 练习3：创建价格表
设计一个三栏价格表，要求：
- 突出推荐的方案（中间列）
- 价格是最突出的元素
- 特性列表清晰可读
- 行动按钮吸引注意

### 练习4：移动端层级优化
将桌面端的复杂布局转换为移动端：
- 重新评估元素重要性
- 调整视觉层级顺序
- 优化间距和对比度

### 练习5：可访问性层级
创建一个既美观又可访问的层级系统：
- 确保对比度符合WCAG标准
- 使用语义HTML标签
- 提供清晰的焦点状态
- 支持屏幕阅读器

## 6.8 总结

视觉层级是UI设计的基石：

1. **核心原则**：使用大小、颜色、对比度、位置、空白创建层级
2. **尺寸系统**：建立一致的尺寸比例，确保和谐
3. **颜色层级**：通过颜色深浅和饱和度传达重要性
4. **空白运用**：给重要内容足够的呼吸空间
5. **避免过度**：不要让所有元素都突出
6. **考虑可访问性**：确保足够对比度
7. **响应式调整**：在不同设备上保持清晰层级

掌握视觉层级设计，你将能够创建清晰、易用、美观的用户界面。
