## 什么是设计系统

### 定义

```
设计系统是一套完整的标准，包括：
- 设计原则
- 设计令牌（Design Tokens）
- 组件库
- 使用指南
- 代码实现
- 最佳实践
```

### 价值

```
✓ 提高一致性
✓ 加快开发速度
✓ 降低维护成本
✓ 改善协作效率
✓ 保证品质
✓ 易于扩展
```

### 知名设计系统

```
Material Design (Google)
- 网站：material.io
- 特点：全面、详细、跨平台

Human Interface Guidelines (Apple)
- 网站：developer.apple.com/design
- 特点：专注用户体验、平台规范

Ant Design
- 网站：ant.design
- 特点：企业级、完整、React生态

Carbon Design System (IBM)
- 网站：carbondesignsystem.com
- 特点：数据密集型应用

Atlassian Design System
- 网站：atlassian.design
- 特点：产品协作工具
```

## 设计令牌

### 什么是设计令牌？

```
设计令牌是设计决策的最小单位：
- 颜色值
- 字体大小
- 间距值
- 阴影效果
- 边框半径
- 动画时长

优势：
✓ 单一数据源
✓ 跨平台一致
✓ 易于维护
✓ 语义化命名
```

```
基础组件（Primitives）：
- Button

布局组件（Layout）：
```

### 组件结构

```
每个组件应包含：

1. 变体（Variants）
   - primary, secondary, tertiary
   - sizes: sm, md, lg
   - states: default, hover, active, disabled

2. 属性（Props）
   - 必需属性

3. 文档

```

### 按钮组件示例

```jsx
// Button.jsx
import React from 'react';
import './Button.css';

const Button = ({
  children,
  variant = 'primary',
  size = 'md',
  disabled = false,
  loading = false,
  icon,
  onClick,
  ...props
}) => {
  const classNames = [
    'button',
    `button--${variant}`,
    `button--${size}`,
    loading && 'button--loading',
    disabled && 'button--disabled',
  ].filter(Boolean).join(' ');

  return (
    <button
      className={classNames}
      onClick={onClick}
      disabled={disabled || loading}
      {...props}
    >
      {icon && <span className="button__icon">{icon}</span>}
      <span className="button__text">{children}</span>
      {loading && <span className="button__spinner" />}
    </button>
  );
};

export default Button;
```

### 组件文档模板

````markdown
# Button 组件

## 描述
通用按钮组件，用于触发操作和事件。

## 导入
```jsx
import { Button } from '@/components';
```

## 基本用法
```jsx
<Button onClick={handleClick}>
  点击我
</Button>
```

## 变体

### Primary（主要）
用于页面最重要的操作。
```jsx
<Button variant="primary">主要按钮</Button>
```

### Secondary（次要）
用于次要操作。
```jsx
<Button variant="secondary">次要按钮</Button>
```

### Tertiary（第三级）
用于低优先级操作。
```jsx
<Button variant="tertiary">文本按钮</Button>
```

## 尺寸
```jsx
<Button size="sm">小按钮</Button>
<Button size="md">中等按钮</Button>
<Button size="lg">大按钮</Button>
```

## 状态

### 禁用
```jsx
<Button disabled>禁用按钮</Button>
```

### 加载中
```jsx
<Button loading>加载中...</Button>
```

## 带图标
```jsx
<Button icon={<IconPlus />}>
  添加
</Button>
```

## Props

| 名称 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| variant | 'primary' \| 'secondary' \| 'tertiary' | 'primary' | 按钮变体 |
| size | 'sm' \| 'md' \| 'lg' | 'md' | 按钮大小 |
| disabled | boolean | false | 是否禁用 |
| loading | boolean | false | 是否加载中 |
| icon | ReactNode | - | 图标元素 |
| onClick | function | - | 点击事件处理 |

## 可访问性
- 使用语义化 `<button>` 元素
- 支持键盘导航（Enter/Space）
- 提供 `:focus-visible` 样式
- 禁用状态使用 `disabled` 属性
- 图标按钮需要 `aria-label`

## 最佳实践
✓ 每个页面最多一个 primary 按钮
✓ 使用清晰的动词（保存、删除、提交）
✓ 危险操作使用确认对话框
✓ 加载状态避免重复提交

❌ 不要只用图标（除非配合 tooltip）
❌ 不要使用过长的文本
❌ 不要嵌套按钮或链接
````

## 版本管理

### 语义化版本

```
版本号格式：主版本.次版本.修订号

主版本（Major）：
- 不兼容的 API 变更
- 重大设计改变
- 示例：1.0.0 → 2.0.0

次版本（Minor）：
- 向后兼容的新功能
- 新增组件
- 示例：1.0.0 → 1.1.0

修订号（Patch）：
- 向后兼容的问题修复
- 样式微调
- 示例：1.0.0 → 1.0.1
```
