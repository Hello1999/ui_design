## 可访问性基础

### 为什么重要？

```
道德层面：
- 每个人都应能访问网页
- 约15%人口有某种残疾

法律层面：
- 许多国家有法律要求
- ADA、Section 508等法规

商业层面：
- 扩大受众群体
- 改善 SEO
- 提升整体用户体验
```

### POUR 原则

```
1. Perceivable（可感知）
   - 信息和UI组件必须能被用户感知
   - 提供文本替代
   - 足够的色彩对比度
   - 可调整文本大小

2. Operable（可操作）
   - 用户必须能操作UI组件和导航
   - 键盘可访问
   - 足够的时间
   - 避免癫痫诱发设计

3. Understandable（可理解）
   - 信息和UI操作必须可理解
   - 可读性
   - 可预测
   - 输入帮助

4. Robust（健壮）
   - 内容必须能被各种用户代理可靠解释
   - 兼容辅助技术
   - 有效的 HTML
```

## WCAG 标准

### 符合等级

```
A级（最低）：
- 基本可访问性
- 必须满足

AA级（推荐）：
- 较好的可访问性
- 大多数组织目标
- 法律通常要求

AAA级（最高）：
- 最佳可访问性
- 不是所有内容都能达到
```

### 关键准则

#### 1.1 文本替代

```html
<!-- ✅ 好：提供alt文本 -->
<img src="logo.png" alt="公司Logo">

<!-- ✅ 装饰性图片：空alt -->
<img src="decoration.png" alt="">

<!-- ❌ 差：缺少alt -->
<img src="logo.png">

<!-- ✅ 复杂图片：详细描述 -->
<img
  src="chart.png"
  alt="2025年销售数据"
  longdesc="#chart-description"
>
<div id="chart-description">
  <p>图表显示2025年季度销售数据...</p>
</div>

<!-- ✅ 图标按钮 -->
<button aria-label="关闭">
  <svg>...</svg>
</button>
```

#### 1.4 辨别性

**色彩对比度：**

```css
/* ✅ AA级：至少 4.5:1 */
.text-aa {
  background: #FFFFFF;
  color: #767676; /* 4.54:1 */
}

/* ✅ AAA级：至少 7:1 */
.text-aaa {
  background: #FFFFFF;
  color: #595959; /* 7.0:1 */
}

/* ❌ 不合格：低于 4.5:1 */
.text-fail {
  background: #FFFFFF;
  color: #CCCCCC; /* 1.6:1 */
}

/* ✅ 大文本（18px+）：AA级 3:1 */
.large-text {
  font-size: 18px;
  background: #FFFFFF;
  color: #959595; /* 3.0:1 */
}
```

#### 2.1 键盘可访问

```html
<!-- ✅ 使用原生可聚焦元素 -->
<button onclick="handleClick()">点击</button>
<a href="/page">链接</a>

<!-- ❌ div不可键盘访问 -->
<div onclick="handleClick()">点击</div>

<!-- ✅ 如果必须用div，添加tabindex和键盘事件 -->
<div
  role="button"
  tabindex="0"
  onclick="handleClick()"
  onkeydown="handleKeyDown(event)"
>
  点击
</div>
```

#### 2.4 导航

```html
<!-- ✅ 跳过导航链接 -->
<a href="#main-content" class="skip-link">
  跳到主内容
</a>

<nav>...</nav>

<main id="main-content">
  <!-- 主要内容 -->
</main>

<style>
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  padding: 8px;
  background: #000;
  color: #fff;
}

.skip-link:focus {
  top: 0;
}
</style>
```

## 键盘导航

### Tab 顺序

```html
<!-- 默认Tab顺序：文档流 -->
<button>第一个</button>
<button>第二个</button>
<button>第三个</button>

<!-- ❌ 不要修改tabindex破坏顺序 -->
<button tabindex="3">第三个</button>
<button tabindex="1">第一个</button>
<button tabindex="2">第二个</button>

<!-- ✅ 使用 tabindex="0" 包含在Tab顺序中 -->
<div role="button" tabindex="0">自定义按钮</div>

<!-- ✅ 使用 tabindex="-1" 排除在Tab顺序外（但可编程聚焦）-->
<div tabindex="-1" id="modal-title">模态框标题</div>
```

### 焦点指示器

```css
/* ❌ 永远不要完全移除outline */
*:focus {
  outline: none; /* 不要这样做！ */
}

/* ✅ 提供清晰的焦点指示器 */
button:focus-visible {
  outline: 2px solid #3B82F6;
  outline-offset: 2px;
}

/* ✅ 自定义焦点样式 */
.custom-focus:focus-visible {
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.5);
  outline: none;
}

/* 聚焦时显示跳过链接 */
.skip-link:focus {
  position: fixed;
  top: 10px;
  left: 10px;
  z-index: 9999;
}
```

### 键盘事件

```javascript
// 处理键盘事件
function handleKeyDown(event) {
  switch(event.key) {
    case 'Enter':
    case ' ': // 空格键
      event.preventDefault();
      handleClick();
      break;
    case 'Escape':
      handleClose();
      break;
    case 'ArrowUp':
      navigatePrevious();
      break;
    case 'ArrowDown':
      navigateNext();
      break;
  }
}

// 焦点陷阱（模态框）
function trapFocus(element) {
  const focusableElements = element.querySelectorAll(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
  );
  const firstElement = focusableElements[0];
  const lastElement = focusableElements[focusableElements.length - 1];

  element.addEventListener('keydown', (e) => {
    if (e.key === 'Tab') {
      if (e.shiftKey) {
        // Shift + Tab
        if (document.activeElement === firstElement) {
          e.preventDefault();
          lastElement.focus();
        }
      } else {
        // Tab
        if (document.activeElement === lastElement) {
          e.preventDefault();
          firstElement.focus();
        }
      }
    }
  });
}
```

## 屏幕阅读器

### 语义化 HTML

```html
<!-- ✅ 使用正确的HTML元素 -->
<header>
  <nav>
    <ul>
      <li><a href="/">首页</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>文章标题</h1>
    <p>内容...</p>
  </article>
</main>

<aside>
  <h2>侧边栏</h2>
</aside>

<footer>
  <p>&copy; 2026</p>
</footer>

<!-- ❌ 全用div（无语义）-->
<div class="header">
  <div class="nav">...</div>
</div>
```

### ARIA 属性

**ARIA 角色：**

```html
<!-- 当HTML元素不足时使用ARIA -->
<div role="button" tabindex="0">自定义按钮</div>
<div role="navigation">...</div>
<div role="alert">重要提示</div>
<div role="status">状态更新</div>
<div role="dialog" aria-modal="true">模态框</div>

<!-- Landmark角色 -->
<div role="banner">网站头部</div>
<div role="main">主内容</div>
<div role="complementary">补充信息</div>
<div role="contentinfo">页脚</div>
```

**ARIA 状态和属性：**

```html
<!-- aria-label：提供标签 -->
<button aria-label="关闭对话框">
  <svg>×</svg>
</button>

<!-- aria-labelledby：引用其他元素作为标签 -->
<div role="dialog" aria-labelledby="dialog-title">
  <h2 id="dialog-title">确认删除</h2>
</div>

<!-- aria-describedby：额外描述 -->
<input
  type="password"
  aria-describedby="password-hint"
>
<p id="password-hint">至少8个字符</p>

<!-- aria-expanded：展开/折叠状态 -->
<button
  aria-expanded="false"
  aria-controls="menu"
  onclick="toggleMenu()"
>
  菜单
</button>
<div id="menu" hidden>...</div>

<!-- aria-current：当前项 -->
<nav>
  <a href="/" aria-current="page">首页</a>
  <a href="/about">关于</a>
</nav>

<!-- aria-hidden：对屏幕阅读器隐藏 -->
<span aria-hidden="true">装饰图标</span>

<!-- aria-live：动态更新通知 -->
<div aria-live="polite" aria-atomic="true">
  <p>搜索到 5 个结果</p>
</div>

<!-- aria-live="assertive" 用于紧急消息 -->
<div role="alert" aria-live="assertive">
  <p>错误：提交失败</p>
</div>
```

**表单可访问性：**

```html
<form>
  <!-- ✅ 关联label和input -->
  <label for="email">邮箱地址</label>
  <input
    type="email"
    id="email"
    name="email"
    required
    aria-required="true"
    aria-invalid="false"
    aria-describedby="email-error"
  >
  <span id="email-error" role="alert">
    <!-- 错误消息 -->
  </span>

  <!-- ✅ 字段集分组 -->
  <fieldset>
    <legend>配送地址</legend>
    <label for="street">街道</label>
    <input type="text" id="street">
    <label for="city">城市</label>
    <input type="text" id="city">
  </fieldset>

  <!-- ✅ 单选按钮组 -->
  <fieldset>
    <legend>选择支付方式</legend>
    <label>
      <input type="radio" name="payment" value="card">
      信用卡
    </label>
    <label>
      <input type="radio" name="payment" value="paypal">
      PayPal
    </label>
  </fieldset>
</form>
```

## 色彩与对比度

### 对比度要求

```
普通文本：
- AA级：4.5:1
- AAA级：7:1

大文本（18px+ 或 14px粗体+）：
- AA级：3:1
- AAA级：4.5:1

UI组件和图形：
- 至少3:1（AA级）
```

### 检查工具

```
在线工具：
- WebAIM Contrast Checker
- Contrast Ratio by Lea Verou
- Accessible Colors

浏览器工具：
- Chrome DevTools（Lighthouse）
- Firefox Accessibility Inspector

Figma插件：
- Contrast
- Stark
```

### 不要只依赖颜色

```html
<!-- ❌ 仅用颜色区分 -->
<div>
  <span style="color: red;">错误</span>
  <span style="color: green;">成功</span>
</div>

<!-- ✅ 颜色 + 图标 + 文字 -->
<div>
  <span class="status-error">
    <svg aria-hidden="true">❌</svg>
    错误
  </span>
  <span class="status-success">
    <svg aria-hidden="true">✓</svg>
    成功
  </span>
</div>

<!-- ✅ 链接不只用颜色区分 -->
<p>
  访问我们的
  <a href="#" style="color: blue; text-decoration: underline;">
    帮助中心
  </a>
</p>
```

## 表单可访问性

### 标签和输入

```html
<!-- ✅ 显式关联 -->
<label for="username">用户名</label>
<input type="text" id="username" name="username">

<!-- ✅ 隐式关联 -->
<label>
  邮箱
  <input type="email" name="email">
</label>

<!-- ✅ aria-label（无可见标签时）-->
<input
  type="search"
  aria-label="搜索文章"
  placeholder="搜索..."
>

<!-- ❌ 仅用placeholder -->
<input type="text" placeholder="用户名">
```

### 错误提示

```html
<div class="form-field">
  <label for="email">邮箱地址 <span aria-label="必填">*</span></label>
  <input
    type="email"
    id="email"
    aria-required="true"
    aria-invalid="true"
    aria-describedby="email-error"
  >
  <span id="email-error" role="alert" class="error-message">
    请输入有效的邮箱地址
  </span>
</div>

<style>
.error-message {
  color: #DC2626;
  font-size: 14px;
  margin-top: 4px;
  display: flex;
  align-items: center;
  gap: 4px;
}

/* 确保图标不影响阅读 */
.error-message::before {
  content: '⚠';
  aria-hidden: true;
}
</style>
```

### 必填字段

```html
<!-- ✅ 多种方式指示必填 -->
<label for="name">
  姓名
  <span aria-label="必填" class="required">*</span>
</label>
<input
  type="text"
  id="name"
  required
  aria-required="true"
>

<!-- ✅ 表单开头说明 -->
<form>
  <p>标有 <span class="required" aria-label="必填">*</span> 的字段为必填</p>
  <!-- 表单字段 -->
</form>
```

## 动画与运动

### 尊重用户偏好

```css
/* 默认：有动画 */
.animated {
  transition: transform 0.3s ease;
}

.animated:hover {
  transform: scale(1.05);
}

/* 用户偏好：减少动画 */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }

  .animated {
    transition: none;
  }

  .animated:hover {
    transform: none;
  }
}
```

### 自动播放

```html
<!-- ❌ 自动播放音频/视频 -->
<video autoplay>

<!-- ✅ 提供控制 -->
<video controls>

<!-- ✅ 自动播放但静音（轮播图等）-->
<video autoplay muted>
```

## 总结

### 核心要点

**基本原则**
- 可感知、可操作、可理解、健壮
- 不要只依赖颜色
- 提供文本替代
- 确保键盘可访问

**WCAG标准**
- 目标：至少AA级
- 对比度：至少4.5:1
- 触摸目标：至少44x44px
- 响应式和可缩放

**实施方法**
- 语义化HTML
- 正确使用ARIA
- 键盘导航支持
- 屏幕阅读器兼容

**测试验证**
- 自动化工具
- 手动测试
- 真实用户反馈
- 持续改进
